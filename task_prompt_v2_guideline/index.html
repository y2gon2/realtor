    RAG v2 메타데이터 보강 작업 가이드라인¶

    작업 목적: rag_refined/ 의 v2 문서에 색인용 메타데이터 5개 필드를 추가하여 rag_v2/ 에 저장한다. 처리 규칙 참조: task_prompt/v2_refine_notes_rag_done.md — 규칙 6 (신규 메타데이터) 및 자기 검증 체크리스트

    1. 이 작업이 하는 일 (범위 명확화)¶

    rag_refined/ 파일들은 이미 v2 품질로 정제된 문서다 (키워드, HyDE 쿼리, 스크립트 정제, Atomic Facts 완료). 이 작업은 콘텐츠를 재작성하지 않는다. 아래만 수행한다:

    1. rag_refined/ 파일을 읽는다
    2. YAML 최상단에 5개의 신규 색인 메타데이터 필드를 추가한다
    3. processing.version 을 "rag-v2-indexed" 로 변경한다
    4. 동일 파일명으로 rag_v2/ 에 저장한다

    수행하지 않는 것: 스크립트 재정제, Atomic Facts 재작성, HyDE 쿼리 재생성 예외 허용: 명백한 오타·깨진 YAML 구조·누락 항목 발견 시 해당 부분에 대해 최소 처리 진행

    2. 추가해야 하는 5개 메타데이터 필드¶

    상세 분류 기준 및 허용 값 목록은 task_prompt/v2_refine_notes_rag_done.md 규칙 6 을 반드시 참조한다.

    # ▼ 아래 5개 필드를 YAML 내 doc_type 바로 아래에 삽입한다
topic_tags:
  - "시황/가격동향"         # 폐쇄형: 허용 목록에서만 선택, 없으면 "기타"
region_tags:
  - "서울/강남구"           # 형식 고정("시도" 또는 "시도/시군구"), 목록 외 지역 추가 가능
asset_type:
  - "아파트"               # 허용 목록에서만 선택
policy_period:
  start: null              # 정책 기간 명시된 경우만 기입, 없으면 null
  end: null
reliability_score:
  level: 3                 # 1~4 정수 (1=공공/정부, 2=전문가, 3=인플루언서, 4=일반사용자)

    3. 출력 YAML 전체 구조 (필드 순서 엄수)¶

    ---
doc_type: "youtube_study_note"
language: "ko"
domain: "real_estate_korea"

topic_tags:
  - "..."
region_tags:
  - "..."
asset_type:
  - "..."
policy_period:
  start: null
  end: null
reliability_score:
  level: N

source:
  platform: "youtube"
  url: "..."
  video_id: "..."
  channel: "..."
  upload_date: "YYYY-MM-DD"
  upload_date_raw: "YYYYMMDD"

processing:
  created_at: "원본 값 유지"
  version: "rag-v2-indexed"      # ← 반드시 이 값으로 변경
  source_script: "원본 값 유지"
  keywords:
    - "..."
---

    4. 세션 시작 시 준비 절차 (1회만 수행)¶

    세션 시작 직후 아래 절차를 순서대로 수행하고 결과를 출력한 뒤 배치 처리로 진입한다.

    4-1. 폴더 및 로그 파일 확인¶

    확인 항목:
- rag_refined/ 존재 여부 및 파일 수
- rag_v2/ 존재 여부 (없으면 생성)
- log/rag_v2.txt 존재 여부 (없으면 생성)

    4-2. 잔여 작업 목록 산출¶

    잔여 목록 = rag_refined/ 파일명 목록  −  rag_v2/ 파일명 목록
(이미 rag_v2/ 에 있는 파일은 재처리하지 않는다)

    산출 후 아래 형식으로 출력한다:

    [준비 완료]
- rag_refined 전체: N개
- rag_v2 처리 완료: M개
- 잔여 처리 대상: K개
- 이번 세션 목표: 최소 50개 처리
- 첫 배치 대상: [파일명1, 파일명2, 파일명3]

    5. 배치 처리 루프 (핵심)¶

    5-1. 1배치 = 3개 파일, 아래 순서로 처리¶

    for each 파일 in [파일1, 파일2, 파일3]:
    1. rag_refined/{파일명}.md 읽기
    2. 문서 내용 분석 → 5개 메타데이터 분류 결정
    3. YAML 재구성 (필드 순서: 규칙 6 스키마 기준)
    4. version → "rag-v2-indexed" 변경
    5. rag_v2/{파일명}.md 로 저장 (원본 rag_refined 파일 유지)

    5-2. 1배치 완료 후 즉시 수행¶

    ① 로그 1줄 기록 — log/rag_v2.txt 에 아래 형식으로 append (덮어쓰기 금지):

    [YYYY-MM-DD HH:MM:SS] <rag_v2 파일 수> / <rag_refined 파일 수> / <특이 사항 - 없으면 생략>

    예시: 

    [2026-03-01 17:00:12] 3 / 727
[2026-03-01 17:04:35] 6 / 727
[2026-03-01 17:09:01] 9 / 727 / --3Fdvx_qwLo: policy_period 기입 (2024-01-01~)

    ② 진행 현황 출력 (사용자에게 표시):

    [배치 N 완료] rag_v2: M개 / 잔여: K개 / 누적 처리: P개
다음 배치: [파일명4, 파일명5, 파일명6]

    ③ 사용자 입력 없이 즉시 다음 배치 시작

    6. 자동 진행 원칙 (중요)¶

    상황 행동
    1배치 정상 완료 즉시 다음 배치 시작, 사용자 확인 요청 금지
    파일 YAML 구조 경미한 이상 최소 수정 후 처리 계속
    파일 읽기 실패 해당 파일 건너뜀 + 로그에 기록, 나머지 2개 처리 후 다음 배치
    토큰 한계 임박 감지 현재 파일 완료 후 로그 기록 → 세션 종료 메시지 출력
    오류·예외 상황 로그 특이사항란에 기록 후 다음 파일로 진행

    절대 금지: 배치 사이에 "계속 진행할까요?", "다음 배치를 시작해도 될까요?" 등의 질문 출력

    7. 파일 조작 제약¶

    허용 금지
    rag_refined/ 파일 읽기 rag_refined/ 파일 삭제·이동·수정
    rag_v2/ 파일 쓰기(생성) 기존 rag_v2/ 파일 덮어쓰기 (이미 완료된 파일)
    log/rag_v2.txt append 로그 파일 덮어쓰기

    rag_refined 원본 보존: 이 작업은 원본을 건드리지 않는다. rag_v2는 별도의 새 파일 세트다.

    8. 세션 목표 및 종료 기준¶

    • 이번 세션 최소 처리 목표: 50개 (이후 지시에 따라 증가 가능)
    • 목표 달성 후에도 토큰 여유가 있으면 계속 진행한다
    • 세션 종료 조건: 토큰 한계 임박 OR 잔여 파일 없음
    • 세션 종료 시 아래 요약 출력:
    [세션 완료]
- 이번 세션 처리: N개
- rag_v2 누적: M개
- rag_refined 잔여: K개
- 로그: log/rag_v2.txt 갱신 완료

    9. 자기 검증 체크리스트 (파일 1개 저장 전 확인)¶

    • [ ] doc_type → "youtube_study_note" (원본 유지)
    • [ ] topic_tags → 허용 목록 내 태그만 사용 (임의 생성 금지, 없으면 "기타")
    • [ ] region_tags → "시도" 또는 "시도/시군구" 형식 (목록 외 지역도 형식만 맞으면 가능)
    • [ ] asset_type → 허용 목록 내 값
    • [ ] reliability_score.level → 1~4 사이 정수
    • [ ] policy_period → 정책 기간 없으면 start: null, end: null
    • [ ] processing.version → "rag-v2-indexed" 로 변경됐는가
    • [ ] 원본 source 필드 내용 (url, channel, upload_date 등) 변경 없이 유지됐는가
    • [ ] 원본 keywords 배열 변경 없이 유지됐는가
    • [ ] 콘텐츠 섹션 (## 핵심 요지, ## 전체 흐름, ## 원자 사실, ## 예상 검색 질문) 변경 없이 유지됐는가

    10. 파일 처리 순서¶

    rag_refined/ 파일을 파일명 알파벳 오름차순으로 정렬하여 순서대로 처리한다. 단, 이미 rag_v2/ 에 존재하는 파일은 건너뛴다.

    참고 파일¶

    파일 용도
    task_prompt/v2_refine_notes_rag_done.md 메타데이터 분류 기준 (규칙 6 전체 + 허용 목록)
    rag_refined/*.md 처리 대상 원본 파일
