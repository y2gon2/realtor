    RAG+LLM vs 순수 LLM 답변 품질 비교 평가 보고서¶

    작성일: 2026-04-04 (최종 업데이트: 2026-04-05) 선행 문서: 00_overview.md, result/summary.md, result_v3/summary.md, opus_result/summary.md 목적: 동일 197개 부동산 질문에 대한 RAG+LLM(Sonnet) vs 순수 LLM(Opus 4.6) 답변의 체계적 비교 평가

    핵심 발견 3가지¶

    1. realestate_v3 연동으로 doc pass율 10.7% → 65.0% 급등 (+54.3%p) - 검색 포인트 63배 확대 (2,921 → 185,365) - YouTube 전문가 콘텐츠가 시황/지역/실전/정책/생활 밀착형 질문을 커버 - Part A 73%, Part C 58%, Part D 57% — 모든 Part에서 균등한 개선

    2. RAG의 강점은 "출처 인용 + 광범위 커버리지"로 확대 - 세금/법률: 법률 조문 인용 기반 정밀 답변 (기존 강점 유지) - 시황/투자: YouTube 전문가 콘텐츠 기반 실전 정보 제공 (신규) - 408건의 realestate_v3 출처 — 채널명/날짜 인용으로 검증 가능성 제공

    3. 할루시네이션 통제 유지 + 레이턴시 30% 감소 - 문서 범위 63배 확대에도 halluc pass 84.3% → 84.8% (미세 개선) - doc pass율 상승 → 재검색 빈도 감소 → 91초 → 64초 단축

    5. 카테고리별 상세 분석 (v3 연동 기준)¶

    5-1. 세금 (A-2, Q16-30) — RAG 최강 카테고리¶

    v3 연동 후에도 세금 카테고리는 RAG의 핵심 강점 영역이다. legal_docs_v2의 법률 조문 + realestate_v3의 세금 전문 채널(AllTax_GAGAM 637건) 콘텐츠가 결합되어 출처 인용의 깊이가 더해졌다.

    RAG 출처 인용의 실질적 가치 — Q16, Q27¶

    Q16: "취득세 얼마 나오나요? 9억짜리 아파트 1주택자요"

    RAG: 법률 문서 기반 3% 세율 계산 + realestate_v3의 세금 전문가 해설 인용 → 사용자가 법적 근거를 직접 추적 가능 + 전문가 경험 기반 보충

    Q27: "재산세 6월 1일 기준이라 잔금일 조절해야 한다면서요?"

    RAG: "잔금일과 등기일 중 더 빠른 날이 취득일" — 문서 기반 비직관적 인사이트 → 이 규칙은 문서에서만 정확히 확인 가능

    발견된 문제 — Q17 문서 시점 불일치¶

    항목 RAG Opus
    양도세 비과세 기준 8억원 이하 (legal_docs_v2 인용) 12억원 이하 (2021.12 개정 반영)

    → 법률 문서의 시점 문제. 문서 upload_date 기반 최신성 검증이 필요.

    Q52 (확정일자): 우선변제권 vs 소액임차인 vs 보증보험 3중 구조 법적 설명 유지 Q53 (경매 보증금): "지역별 소액보증금 기준 상이" — false precision 방지

    5-3. 매매/시세 (A-1, Q1-15) — v3 연동 후 대폭 개선¶

    v3 연동 전: doc pass 낮음, 대부분 "확인할 수 없습니다" 거부 v3 연동 후: Part A 전체 doc pass 73% — 시황 분석 YouTube 콘텐츠(maeburitv 1,004건, weolbu_official 1,064건)가 시장 동향/투자 전략 질문에 대한 답변 근거를 제공

    6-1. RAG 할루시네이션 통제: 84.8% pass¶

    시세 예측/타이밍 ~30% 문서로 답변 불가 P2: 파라메트릭 보완
    초세부 지역 정보 ~25% 해당 콘텐츠 부재 데이터 확장
    정부 정책 최신 ~20% 문서 시점 이후 변경 P4: 시점 검증
    절차적 세부 사항 ~15% 실무 디테일 부족 데이터 확장
    기타 ~10% 생활 정보 데이터 확장

    8-3. 향후 우선순위¶

    순위 작업 기대 효과 소요
    P1 document fail 시 fallback 전략 잔여 35% 중 ~10%p 추가 2-3일
    P2 RAG + 파라메트릭 지식 하이브리드 시세 예측 질문 대응 3-5일
    P3 멀티턴 컨텍스트 구현 Part D 대화 품질 향상 2-3일
    P4 문서 시점 검증 (upload_date 면책) Q17류 오류 방지 1일
    P5 답변 프롬프트 개선 (Opus 벤치마킹) 사용자 만족도 향상 2-3일

    P1: document fail 시 fallback 검색 전략¶

    현재 파이프라인은 3개 컬렉션을 동시에 검색하지만, grade_documents가 fail 판정 시 동일한 3개 컬렉션에서 재검색한다. fail 질문의 상당수는 검색 범위가 아닌 검색 질의의 표현 문제이므로, 단계적 전략이 필요하다.

    [현재]
검색 (onto+legal+realestate) → fail → rewrite → 재검색 (같은 3개 컬렉션)

[제안]
검색 (onto+legal+realestate) → fail → rewrite → 재검색
    → 여전히 fail → realestate 단독 확장 검색 (limit 2배, 메타데이터 필터 완화)
    → 여전히 fail → "확인할 수 없습니다" + 관련 주제 제안
    • realestate 단독 확장 검색: limit을 5→10으로 늘리고, topic_tags/region_tags 기반 필터를 추가하여 검색 recall 확보
    • Harvey AI의 "Recall over Precision + LLM 필터링" 전략 적용
    • 기대 효과: 잔여 35% fail 중 초세부 지역(~25%)과 절차적 세부(~15%) 일부 해소, doc pass 65% → ~75%

    P2: RAG + 파라메트릭 지식 하이브리드 답변¶

    잔여 fail의 ~30%는 "시세 예측/타이밍" 질문으로, 어떤 문서로도 답변 불가하다. 이 경우 LLM의 파라메트릭 지식을 조건부로 허용하되, 명확히 구분 표시한다.

    [문서 근거 기반 답변]
"취득세는 주택 가격에 따라 1-3%가 적용됩니다. [1] 법률: 제1편 주택의 취득과 관련된 세금"

[일반 지식 기반 보충] ← doc fail 시에만 활성화
"※ 아래 내용은 검색된 문서에 포함되지 않은 일반 정보입니다:
 - 조정대상지역 2주택 시 8% 중과세율이 적용될 수 있습니다.
 - 정확한 적용 여부는 세무사 상담을 권장합니다."
    • Retrieval Confidence Scorer: CE 점수 + CRAG grade 기반 3단계 (HIGH/MEDIUM/LOW)
    • HIGH (CE ≥ 0.7): 확신 있는 답변 + 출처
    • MEDIUM (0.3 ≤ CE < 0.7): 답변 + "참고 수준" 면책
    • LOW (CE < 0.3): 파라메트릭 보완 + "일반 정보" 구분 표시
    • Opus의 포괄성과 RAG의 신뢰성을 동시에 확보
    • 기대 효과: 시세 예측 질문에서도 유용한 프레임워크 제공, 사용자 만족도 대폭 향상

    P3: 멀티턴 컨텍스트 구현¶

    Part D 테스트에서 RAG의 구조적 한계: 각 턴을 독립 질의로 처리하여 대화 맥락이 완전히 단절됨. Opus는 동일 세트에서 12건+ 명시적 이전 턴 참조를 보여줌.

    [현재] 각 턴 독립 처리
Q3: "신촌 원룸 시세?" → 검색 + 답변
Q4: "보증금 1000 월세 50 적당?" → 독립 검색 (Q3의 "신촌" 컨텍스트 무시)

[제안] 대화 히스토리 기반 코레퍼런스 해소
Q3: "신촌 원룸 시세?" → 검색 + 답변 + 컨텍스트 저장
Q4: "보증금 1000 월세 50 적당?" → 코레퍼런스 해소: "신촌 지역에서 보증금 1000 월세 50 적당?"
                                → 컨텍스트 포함 검색 + 답변
    • 00_overview.md에서 설계된 코레퍼런스 해소 메커니즘 활용:
    • 슬라이딩 윈도우 5턴
    • 코레퍼런스 마커: "그거", "이거", "그건", "그럼", "거기" 감지
    • 도메인 키워드 부재 + 짧은 질의 → LLM 기반 코레퍼런스 해소 발동
    • 기대 효과: Part D halluc pass 84% 유지하면서 doc pass 57% → ~70% 개선

    P4: 문서 시점 검증 체계¶

    Q17 사례(양도세 비과세 기준 8억 vs 12억)에서 드러난 문제: 문서가 존재해도 최신이 아닐 수 있음. 사용자는 RAG의 출처 인용을 "최신 정보"로 신뢰할 가능성이 높다.

    해결 방안: 1. 답변 시 문서의 upload_date 자동 표시: "이 정보는 {날짜} 기준입니다" 2. 2년 이상 경과 문서에 경고: "이 정보는 {년}년 기준이며, 현행 법령과 다를 수 있습니다" 3. 동일 주제 최신 문서 우선 랭킹 (upload_date 기반 recency bias)

    • 수정 파일: gen_prompts.py (프롬프트에 시점 인식 규칙 추가), citation_formatter.py (출처에 날짜 강조)
    • 기대 효과: false precision 방지, 법률 변경 시 사용자 오해 감소

    P5: 답변 생성 프롬프트 개선 (Opus 벤치마킹)¶

    Opus 답변의 강점을 RAG 프롬프트에 흡수: - 구조화된 답변 포맷: 표, 체크리스트, 단계별 가이드 (Opus 스타일) - 실용적 행동 지침: "지금 당장 할 수 있는 행동 3가지" 섹션 - 맥락 적응: 질문자의 상황(신혼부부, 사회초년생, 투자자)에 맞는 어조 - 후속 제안: Opus의 41% 후속 제안처럼, 관련 주제 2-3개 자발적 제안

    현재 GENERATE_SYSTEM_PROMPT에 추가: 

    - 답변은 표, 비교 차트, 단계별 가이드 형태로 구조화하세요
- 답변 끝에 "추가로 궁금하실 수 있는 관련 주제" 2-3개를 제안하세요
- 질문자의 상황(초보/전문가, 실거주/투자)을 추론하여 어조를 조절하세요

    • 기대 효과: 답변 사용자 만족도 향상, Opus 대비 "완전성"과 "실용성" 차원 격차 축소
