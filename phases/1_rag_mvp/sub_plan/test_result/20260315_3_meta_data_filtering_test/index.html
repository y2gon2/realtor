    현상 빈도 설명
    Hybrid top-1 score 1.0 달성 2/5 (40%) 필터로 범위가 좁아지면 Dense+Sparse 상위 문서가 겹칠 확률 증가
    Hybrid 고유 결과 품질 양호 전세사기특별법, 청약 제도 개편 등 키워드 기반 실무 문서 추가 발견
    Hybrid 노이즈 낮음 필터가 범위를 한정하여 BM25 키워드 노이즈가 줄어듦

    필터 + Hybrid 조합이 효과적: 필터로 도메인을 좁히면 BM25 노이즈가 줄고, Dense+Sparse 겹침이 늘어 RRF 점수도 상승.

    4-4. chunk_type별 필터링 가이드¶

    사용 시나리오 권장 chunk_type 이유
    자연어 질의 (RAG 파이프라인) hyde 우선 자연어 질문과 가장 높은 매칭 점수
    문서 전체 맥락 파악 summary 종합적 정보 — LLM 컨텍스트에 적합
    구체적 팩트 확인 atomic_fact 개별 사실 기반 — 정확도 중시
    필터 없이 검색 혼합 (필터 없음) hyde가 top에 올라오되 다양한 관점 확보

    5. 확인 포인트 점검 결과¶

    확인 포인트 결과
    필터 적용 시 결과가 해당 카테고리로 정확히 한정되는가 통과 — 9개 테스트 모두 100% 정확
    필터링 후에도 시맨틱 유사도 순서가 합리적인가 통과 — 모든 테스트에서 score 내림차순 유지
    복합 필터 시 결과 수가 지나치게 적지 않은가 통과 — 모든 복합 필터에서 5건 반환 (데이터 커버리지 양호)

    6. 개선 권고사항¶

    우선순위 항목 설명
    P1 RAG 파이프라인에서 자동 필터 추출 사용자 질의에서 지역명, 자산 타입, 채널명 등을 LLM이 자동 추출하여 필터로 변환하면 정밀도 향상 가능
    P2 channel 필터 사용 시 주의 channel 필터는 score 하락폭이 크므로, 특정 채널 콘텐츠를 명시적으로 요청하는 경우에만 사용 권장
    P2 필터 + Hybrid 조합 기본 전략 도입 필터가 BM25 노이즈를 줄여주므로, 메타데이터 필터 적용 시 Hybrid 검색을 기본으로 사용 권장
    P3 빈 topic_tags/channel 데이터 보강 일부 문서에서 topic_tags, channel이 빈 값. 태깅 커버리지 확대 필요
