    Phase 2A Quick Wins — 종합 계획 및 연구 배경¶

    작성일: 2026-03-28 선행 문서: 7_phase2_comprehensive_evaluation.md, 09_step5_rerank_results.md 관련 코드: codes/query/pipeline.py, codes/query/config.py, codes/ontology/ 목적: P@3 68.0% → 72-75% 달성을 위한 3가지 Quick Win의 종합 계획 및 이론적 배경

    1. 현재 상태와 목표¶

    1-1. 현재 시스템 구성¶

    [사용자 질의]

    ▼

    2. 핵심 개념 설명¶

    2-1. Cross-Encoder 점수 퓨전이란?¶

    비유: 대학교 입시에서 수능 점수(RRF 점수)와 면접 점수(CE 점수)를 합산하는 것과 비슷하다. α는 "수능을 얼마나 중시할 것인가"를 결정하는 비중이다.

    현재 시스템은 검색 결과의 최종 순위를 정할 때, 두 가지 점수를 합산한다:

    final_score = α × rrf_rank_score + (1 - α) × ce_score
    • RRF rank score: Dense+Sparse 검색에서의 순위를 0~1로 정규화한 점수
    • 1위면 1.0에 가깝고, 20위면 0.0에 가까움

    • "키워드와 의미가 모두 비슷한 문서"를 찾음

    • CE score: Cross-Encoder가 (질의, 문서) 쌍을 직접 읽고 매긴 관련성 점수

    • 0~1 사이 값 (높을수록 관련성 높음)

    • "이 질의와 이 문서가 정말 관련이 있는가?"를 정밀 판단

    • α (alpha): 두 점수의 비중을 결정하는 가중치

    • α=0.7이면 RRF 70%, CE 30% → RRF 신뢰
    • α=0.3이면 RRF 30%, CE 70% → CE 신뢰

    현재 문제: α=0.5로 고정되어 있어, 모든 질의에 동일한 비중이 적용됨.

    2-2. CE 양극화(Bimodal Distribution)란?¶

    비유: 면접관이 "합격" 또는 "불합격"만 판정하고, "조건부 합격" 같은 중간 판정을 내리지 않는 상황. 이러면 1점 차이로 탈락하는 사람이 정확히 평가받지 못한다.

    Cross-Encoder 모델(bge-reranker-v2-m3-ko)의 점수 분포:

    점수 분포 (500개 질의 × 20 후보 = 10,000개 점수):

0.0 ████████████████████████████████████  70%
0.1 ██                                     3%
0.2 █                                      2%
0.3 █                                      1%
0.4                                        0.5%
0.5                                        0.5%
0.6 █                                      2%
0.7 ██                                     3%
0.8 ████                                   8%
0.9 █████                                 10%

    왜 이런 분포가 생기는가?

    CE 모델은 Binary Cross-Entropy (BCE) loss로 학습된다. 이 손실 함수는 "관련 있음(1)" 또는 "관련 없음(0)"으로만 학습하기 때문에, 모델이 자연스럽게 0 또는 1 근처로 점수를 밀어넣는다.

    왜 문제인가?

    극단 구어체 질의 (Set B, "나라에 돈 내야 되나")에서: - 정답 문서 "취득세"의 CE 점수: 0.01 (관련 없음으로 판정) - 오답 문서 "공과금"의 CE 점수: 0.03 (약간 더 높음) - 결과: CE가 정답을 밀어내고 오답을 올림 → P@3 하락

    2-3. Reciprocal Rank Fusion (RRF)이란?¶

    비유: 여러 선생님이 각각 학생을 순위 매긴 뒤, "순위가 높을수록 더 많은 점수"를 주는 방식으로 합산하는 것. 한 선생님이 1등으로 뽑은 학생은 다른 선생님이 5등으로 뽑아도 높은 점수를 받는다.

    RRF_score(doc) = Σ weight_i × 1/(k + rank_i + 1)
# k = 60 (논문 원본 최적값)
# rank_i = i번째 검색 결과에서의 순위 (0-based)

    RRF의 핵심 장점: 서로 다른 스케일의 점수를 순위만으로 합산할 수 있다. - Dense 검색: 코사인 유사도 0~1 - Sparse 검색: BM25 점수 0~수십 - 두 점수를 직접 합산하면 스케일이 맞지 않지만, 순위로 변환하면 공정한 합산 가능

    2-4. 온톨로지 Alias란?¶

    비유: 전화번호부에서 "홍길동"을 찾을 때, "길동이", "길동 형", "Hong" 같은 별명으로도 찾을 수 있게 등록해놓는 것.

    온톨로지 엔트리 예시:

    {
  "term": "가산세",           // 정식 이름
  "aliases": [               // 별명 목록 (= 이 이름으로도 검색 가능)
    "추가세금",               // 유의어
    "세금 벌칙금",            // 풀어쓴 표현
    "세금 늦게 내면 벌금",     // 구어체
    "세금 가산세 얼마나 붙어"   // 질문형 구어체
  ]
}

    현재 문제: "영끌", "깡통전세", "줍줍" 같은 인터넷 슬랭이 aliases에 등록되어 있지 않다.

    검색 시 alias가 term + description과 합쳐져 하나의 벡터로 임베딩되므로, alias가 없으면 슬랭 질의가 해당 엔트리와 매칭되지 않는다:

    임베딩 텍스트:
  가산세
  추가세금 | 세금 벌칙금 | ... | 세금 가산세 얼마나 붙어
  세법에서 규정하는 의무의 성실한 이행을 확보하기 위하여...

→ 이 텍스트가 1024차원 벡터 하나로 변환됨
→ 사용자 질의 "영끌"과의 유사도가 매우 낮음 (alias에 없으므로)

    3. 연구 근거¶

    3-1. α 최적화 관련¶

    연구 핵심 발견 우리 시스템 적용
    Bruch et al. (ACM TOIS 2023) Convex combination이 RRF보다 모든 데이터셋에서 nDCG 우위. α 튜닝은 10-50개 라벨 질의만으로 도메인 최적값 도달 500개 테스트셋으로 α 그리드 서치 실행
    Adaptive RAG (NAACL 2024) 질의 복잡도별 다른 전략이 고정 전략보다 우수 SIMPLE/REWRITE/DECOMPOSE 분기에 α를 연동
    Niculescu-Mizil & Caruana (ICML 2005) Isotonic Regression이 bimodal 분포 보정에 Platt Scaling보다 우수 (18/20 테스트) Phase 2B에서 적용 예정

    3-2. CE 양극화 관련¶

    연구 핵심 발견 우리 시스템 적용
    Sentence-Transformers Issue #2874 CE의 bimodal 점수는 BCE loss 학습의 본질적 특성 (버그 아님) 구조적 해결 필요
    Score Distribution Research (Scott Roy) "bimodal shape disappears after calibration" Per-query min-max 정규화로 1차 보정
    AcuRank (2025) Uncertainty 기반 적응적 리랭킹 — 리랭커 호출 25% 감소 + 동등 성능 구어체 질의에서 CE 조건부 스킵

    3-3. 슬랭 alias 확장 관련¶

    연구 핵심 발견 우리 시스템 적용
    LLMs4OL (2024-2025) Hybrid pipeline(LLM + embeddings)으로 ontology expansion F1 0.82-0.88 LLM few-shot으로 슬랭→전문용어 매핑
    "Toward Informal Language Processing" (2024) GPT-4 슬랭 감지 F1: zero-shot 84.4%, fine-tuned 90.1% Few-shot 프롬프트 설계
    Text Enrichment (2024) alias를 chunk text에 삽입 시 Banking77에서 -3.45%p 하락 Set A 회귀 모니터링 필수

    4. 실행 순서와 의존성¶

    작업 1-A: α 그리드 서치 (2시간)
    │ ← 최적 α 범위 확정
    ▼
작업 1-B + 2-A: 조건부 α + CE 우회 (0.5일, 병행 가능)
    │
    ├── 작업 2-B: CE Min-Max 정규화 (포함)
    │
    ▼

    ▼
