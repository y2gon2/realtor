    확장 벤치마크 결과 — 설정 C / E / F 비교 (500개 질의)¶

    테스트 일시: 2026-03-28 환경: DGX Spark, rag-embedding 컨테이너 (BGE-M3 + Qdrant v1.17.0) Cross-Encoder 모델: dragonkue/bge-reranker-v2-m3-ko (sentence-transformers CrossEncoder) 테스트 스크립트: codes/embedding/search_test_phase2_v2.py + codes/query/test_queries.py 질의 소스: data/test_question_v0.md — 500개 (A:150, B:50, C:100, D:100, E:100)

    1. 종합 비교표¶

    1-1. 전체 500개 질의¶

    설정 벡터 구성 퓨전/리랭킹 P@3 정답/총 Avg Top-1 Latency p95
    C: D+S RRF Dense + Sparse RRF 65.8% 329/500 0.7885 65ms
    E: D+S RRF + CE Dense + Sparse RRF → Cross-Encoder 62.6% 313/500 0.1907* 146ms
    F: 3-Way RRF Dense + Sparse + ColBERT RRF (3-way) 65.4% 327/500 1.2022** 221ms

    Setting E의 Avg Top-1은 Cross-Encoder 스코어(0~1, 이진적 분포 특성). *Setting F의 Top-1 스코어는 RRF 스케일(0~3 범위)이므로 Setting C(0~2)와 직접 비교 불가.

    1-2. 세트별 비교¶

    세트 Setting C P@3 Setting E P@3 Setting F P@3 최적 설정
    A (150) 79.3% 76.0% 78.0% C
    B (50) 46.0% 38.0% 46.0% C/F
    C (100) 81.0% 79.0% 82.0% F
    D (100) 50.0% 47.0% 53.0% F
    E (100) 56.0% 54.0% 52.0% C
    전체 65.8% 62.6% 65.4% C

    1-3. 이전 25개 질의 결과와의 비교¶

    설정 이전 P@3 (25개) 현재 P@3 (500개) 변화
    C 92% 65.8% -26.2%p
    E 76% 62.6% -13.4%p
    F 88% 65.4% -22.6%p

    P@3 전반적 하락은 500개 질의에 극단 구어체(Set B, D)와 인터넷 슬랭(Set E)이 포함되었기 때문이며, 순수 벡터 검색의 한계를 반영한다.

    2. Setting C (Dense+Sparse RRF) — Baseline 상세¶

    2-1. 세트별 상세¶

    세트 P@3 정답/총 Avg Top-1 p95
    A (150) 79.3% 119/150 0.8294 61ms
    B (50) 46.0% 23/50 0.7096 62ms
    C (100) 81.0% 81/100 0.7942 62ms
    D (100) 50.0% 50/100 0.7270 64ms
    E (100) 56.0% 56/100 0.7824 65ms
    전체 65.8% 329/500 0.7885 65ms

    2-2. 강점¶

    • 전체 최고 P@3 (65.8%): 세 설정 중 가장 높은 정밀도
    • 최저 레이턴시 (65ms p95): Setting E(146ms), Setting F(221ms) 대비 절반 이하
    • 안정적 Top-1 분포: Avg Top-1 0.7885로 세트 간 편차가 적음 (0.71~0.83)

    2-3. 약점¶

    • 구어체 질의(Set B: 46%, Set D: 50%)에서 절반 이상 실패
    • LLM Query Rewriting 없이는 극단 구어체 질의 해결 불가

    3. Setting E (Cross-Encoder Rerank) — 회귀 분석¶

    3-1. 세트별 상세¶

    세트 P@3 정답/총 Avg CE Top-1 p95
    A (150) 76.0% 114/150 0.2220 137ms
    B (50) 38.0% 19/50 0.2432 138ms
    C (100) 79.0% 79/100 0.1913 130ms
    D (100) 47.0% 47/100 0.2039 134ms
    E (100) 54.0% 54/100 0.0930 146ms
    전체 62.6% 313/500 0.1907 146ms

    3-2. Setting C 대비 회귀 분석¶

    세트 Setting C Setting E 변화 분석
    A (150) 79.3% 76.0% -3.3%p CE가 RRF의 다수결 결과를 뒤집어 일부 정답 소실
    B (50) 46.0% 38.0% -8.0%p 가장 큰 하락. 구어체에서 CE 스코어가 전반적으로 낮음
    C (100) 81.0% 79.0% -2.0%p 소폭 하락
    D (100) 50.0% 47.0% -3.0%p 구어체에서 CE가 약함
    E (100) 56.0% 54.0% -2.0%p 소폭 하락
    전체 65.8% 62.6% -3.2%p 전 세트에서 회귀 발생

    3-3. CE 스코어 분포 문제¶

    관찰 설명
    Avg CE Top-1: 0.1907 전체 평균이 0.2 이하로 매우 낮음
    이진적 분포 대부분의 스코어가 0에 가깝거나 1에 가까움. 중간 점수가 드뭄
    Set E Avg CE: 0.0930 인터넷 슬랭 질의에서 CE 스코어가 거의 0에 수렴
    Near-zero 스코어 다수 CE가 "관련 없음"으로 판정하는 후보가 대부분

    3-4. 판정¶

    Cross-Encoder는 전 세트에서 회귀를 일으킨다 (65.8% → 62.6%, -3.2%p). 특히 Set B에서 -8.0%p로 가장 큰 하락. CE 스코어 분포가 이진적이고 평균이 0.19로 매우 낮아, RRF 후보의 미세한 순위 차이를 감별하는 데 실패한다. 현재 데이터셋에서는 프로덕션 채택 불가.

    4. Setting F (3-Way RRF: Dense+Sparse+ColBERT) — 상세 분석¶

    4-1. 세트별 상세¶

    세트 P@3 정답/총 Avg Top-1 p95
    A (150) 78.0% 117/150 1.2725 134ms
    B (50) 46.0% 23/50 1.1163 168ms
    C (100) 82.0% 82/100 1.1970 154ms
    D (100) 53.0% 53/100 1.1318 162ms
    E (100) 52.0% 52/100 1.1932 221ms
    전체 65.4% 327/500 1.2022 221ms

    4-2. Setting C 대비 비교¶

    세트 Setting C Setting F 변화 분석
    A (150) 79.3% 78.0% -1.3%p 근소한 하락
    B (50) 46.0% 46.0% 0.0%p 동일
    C (100) 81.0% 82.0% +1.0%p ColBERT가 크로스도메인에서 소폭 기여
    D (100) 50.0% 53.0% +3.0%p ColBERT가 구어체에서 소폭 기여
    E (100) 56.0% 52.0% -4.0%p 인터넷 슬랭에서 ColBERT 투표가 부정적
    전체 65.8% 65.4% -0.4%p 실질적 차이 없음

    4-3. 레이턴시 대비 효과¶

    지표 Setting C Setting F 증가율
    p95 레이턴시 65ms 221ms +240%
    P@3 65.8% 65.4% -0.4%p

    ColBERT는 레이턴시를 3.4배 증가시키면서도 P@3를 개선하지 못한다. Set C(+1%p), Set D(+3%p)에서의 소폭 개선은 Set A(-1.3%p), Set E(-4%p)의 하락으로 상쇄된다.

    4-4. 판정¶

    3-Way RRF는 Setting C와 실질적으로 동일한 P@3 (65.4% vs 65.8%)를 보이지만, 레이턴시가 65ms → 221ms로 3.4배 증가한다. ColBERT의 3번째 voter 효과가 일부 세트에서 미세하게 긍정적이나, 전체적으로 비용 대비 효과가 부족하다.

    5. 결론 — Setting C 최적 유지¶

    5-1. 핵심 발견¶

    순위 발견 근거
    1 Setting C가 최적 P@3 65.8% (최고), p95 65ms (최저)
    2 CE Rerank(E)는 회귀 유발 -3.2%p 하락, CE 스코어 분포 문제
    3 3-Way RRF(F)는 비용 대비 효과 부족 -0.4%p 차이에 레이턴시 3.4배
    4 구어체 질의가 핵심 병목 Set B(46%), D(50%) — 세 설정 모두 유사하게 실패

    5-2. 설정별 최종 판정¶

    설정 P@3 p95 판정 비고
    C (D+S RRF) 65.8% 65ms 현행 유지 (프로덕션) 최고 P@3 + 최저 레이턴시
    E (D+S RRF + CE) 62.6% 146ms 미채택 전 세트 회귀, CE 스코어 분포 문제
    F (3-Way RRF) 65.4% 221ms 미채택 P@3 동등, 레이턴시 3.4배 증가

    5-3. 권장 사항¶

    우선순위 작업 근거
    즉시 Setting C 유지 (D+S RRF) 500개 질의에서 재확인된 최고 성능
    P1-a Query Rewriting (LLM 질의 변환) 구어체 질의(B+D: ~48%)가 세 설정 모두에서 공통 병목
    P1-b CE를 Query Rewriting 후 2차 리랭커로 재평가 질의 품질 개선 후 CE 효과 재검증
    보류 3-Way RRF(F)는 fallback으로 보존 ColBERT 필요 시 사용 가능

    6. 성과 궤적 (Phase 2 누적)¶

    Phase 2 검색 성능 개선 궤적:

── 25개 질의 (2026-03-27) ──
Precision@3:
  Baseline (Phase 2 초기):           80%
  + Contextual Retrieval (P0-a):      90%   (+10%p)
  + BGE-M3 D+S RRF (P0-b, C):       92%   (+2%p)
  + Cross-Encoder (E):               76%   (-16%p)  ← 미채택
  + 3-Way RRF (F):                  88%   (-4%p)   ← 미채택

── 500개 질의 확장 (2026-03-28) ──
Precision@3:
  Setting C (D+S RRF):              65.8%           ← 현행 유지
  Setting E (D+S RRF + CE):         62.6%  (-3.2%p) ← 미채택
  Setting F (3-Way RRF):            65.4%  (-0.4%p) ← 미채택

핵심 병목:
  정규/크로스도메인 (A+C):            80.0%  ← 벡터 검색 유효
  구어체/슬랭 (B+D+E):              51.6%  ← LLM Query Rewriting 필수
