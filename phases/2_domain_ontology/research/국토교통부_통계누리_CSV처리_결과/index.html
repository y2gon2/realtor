    국토교통부 통계누리 CSV 처리 결과¶

    작성일: 2026년 3월 22일 데이터 출처: 국토교통부 통계누리 (stat.molit.go.kr) — 사용자 수동 다운로드 저장 위치: data/domain_ontology/raw_data/국토교통_통계누리/

    1. 개요¶

    국토교통부 통계누리는 JS 렌더링 및 방화벽으로 WebFetch 자동 수집 불가. 사용자가 직접 사이트에서 6개 CSV 파일을 다운로드 후 제공.

    처리 방식: 대용량 시계열 CSV → Python 집계 스크립트 → RAG용 요약 Markdown 4개 생성

    CSV 파싱 특이사항¶

    통계누리 CSV 다운로드 시 숫자의 천단위 쉼표가 컬럼 구분자와 혼동되어 숫자가 여러 컬럼으로 분리됨.

    예: 535,971 → ['535', '971'] (두 개의 셀로 분리)
    7,546  → ['7', '546']
    286    → ['286']  (단독 3자리는 그대로)

    해결 전략: - 단일 숫자 컬럼 파일 (지역별 인허가, 미분양 등): 나머지 모든 숫자 토큰을 이어붙여 하나의 정수로 파싱 - 다중 숫자 컬럼 파일 (착공현황): 첫 번째 숫자 토큰만 파싱 (1~2자리 + 3자리 = 천단위 분리 패턴 감지)

    4. RAG 활용 전략¶

    현재 구현 (Phase 0)¶

    CSV 원본 (138,824행)
    ↓ generate_summaries.py (Python 집계)
요약 Markdown 4개 (약 12KB)
    ↓
기존 RAG 텍스트 파이프라인 색인
    ↓
LLM 컨텍스트 직접 삽입 → 전국/시도 수준 Q&A 대응

    대응 가능 질문 유형: - "2024년 전국 주택 인허가 실적은?" - "2023년 미분양이 가장 많았던 시점은?" - "준공 후 미분양 추이는 어떻게 됩니까?" - "수도권과 지방 착공 추이 비교"

    대응 불가 질문 유형: - "강남구 2024년 미분양은 몇 호?" → 시·군·구 세부 데이터 필요 - "경기도 각 시군 착공 현황" → 시도 이하 세부 데이터 필요

    5. 처리 스크립트¶

    위치: data/domain_ontology/raw_data/국토교통_통계누리/generate_summaries.py

    실행 방법: 

    cd data/domain_ontology/raw_data/국토교통_통계누리
python3 generate_summaries.py

    의존성: Python 3 표준 라이브러리만 사용 (csv, codecs, collections) — pandas 불필요

    재실행 조건: CSV 파일 업데이트 시 스크립트 재실행으로 Markdown 갱신
