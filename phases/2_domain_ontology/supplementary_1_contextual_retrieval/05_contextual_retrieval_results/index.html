    Contextual Retrieval 적용 결과 리포트 (500개 질의 확장)¶

    테스트 일시: 2026-03-28 환경: DGX Spark, rag-embedding 컨테이너 (BGE-M3 + Qdrant v1.17.0) 테스트 스크립트: codes/embedding/search_test_phase2_v2.py + codes/query/test_queries.py 질의 소스: data/test_question_v0.md — 500개 (A:150, B:50, C:100, D:100, E:100)

    1. 종합 지표 비교 (기존 25개 → 500개 확장)¶

    1-1. 이전 결과 (25개 질의, 2026-03-27)¶

    지표 Before (prefix 없음) After (prefix 적용) 변화
    Precision@3 80% 90% +10%p
    Avg Ontology Top-1 Score 0.6112 0.6299 +0.0188
    Avg Legal Top-1 Score 0.6155 0.6476 +0.0320

    1-2. 500개 질의 결과 — Setting C Baseline (Dense+Sparse RRF)¶

    지표 기존 25개 500개 확장 변화
    Precision@3 92% 65.8% -26.2%p
    정답 수 23/25 329/500 —
    Avg Top-1 Score 0.7907 0.7885 -0.0022
    Latency p95 48ms 65ms +17ms

    P@3 하락은 예상된 결과: 500개 질의에는 극단적 구어체(Set B, D)와 혼합형(Set E)이 포함되어 있어, 순수 벡터 검색의 한계가 노출된다.

    2. 세트별 상세 결과 — Setting C Baseline¶

    세트 설명 질의 수 P@3 정답/총 Avg Top-1 p95
    A 정규 질의 (formal) 150 79.3% 119/150 0.8294 61ms
    B 극단 구어체 (extreme colloquial) 50 46.0% 23/50 0.7096 62ms
    C 크로스 도메인 (formal terms) 100 81.0% 81/100 0.7942 62ms
    D 극단 구어체 (colloquial) 100 50.0% 50/100 0.7270 64ms
    E 혼합형 (인터넷 슬랭 포함) 100 56.0% 56/100 0.7824 65ms
    전체 — 500 65.8% 329/500 0.7885 65ms

    3. 질의 유형별 성패 분석¶

    3-1. 성공 구간 — 정규/준정규 질의¶

    세트 P@3 분석
    A (정규 질의) 79.3% 전문용어 또는 반전문 표현이 포함된 질의. Dense+Sparse RRF가 키워드 매칭과 의미 매칭을 동시에 활용하여 양호한 성능
    C (크로스 도메인) 81.0% 정규 용어가 포함된 크로스 도메인 질의. "재건축 + 양도세", "청약 + 대출" 등 두 도메인을 결합하더라도 정규 용어가 있으면 검색 성공률이 높음

    3-2. 실패 구간 — 극단 구어체 / 인터넷 슬랭¶

    세트 P@3 분석
    B (극단 구어체) 46.0% "나라에 돈 내야 되나", "몇 프로까지 빌려주는지" 등 전문용어가 전혀 없는 일상어 질의. Sparse 벡터의 키워드 매칭이 실패하고, Dense 벡터만으로는 의미적 거리가 큼
    D (극단 구어체) 50.0% Set B와 유사한 패턴. 구어체 표현과 전문 용어 사이의 어휘 격차(lexical gap)가 핵심 병목
    E (혼합형) 56.0% 인터넷 슬랭("영끌", "갭투", "로또 아파트" 등)이 포함된 질의. 슬랭 표현이 임베딩 공간에서 전문용어와 거리가 멀어 검색 품질 저하

    4. 핵심 발견¶

    4-1. 순수 벡터 검색은 정규/준정규 질의에서 효과적¶

    • Set A (79.3%), Set C (81.0%): 정규 용어가 포함된 질의에서 Dense+Sparse RRF가 안정적으로 작동
    • Avg Top-1 Score도 0.79~0.83 범위로 양호

    4-2. 구어체 질의가 핵심 병목¶

    • Set B (46%), Set D (50%): 전문용어 없이 일상어로만 구성된 질의에서 절반 이상 실패
    • "나라에 돈 내야 되나" ↔ "취득세", "몇 프로까지 빌려주는지" ↔ "LTV" 사이의 의미적 거리를 벡터만으로 극복 불가

    4-3. LLM Query Rewriting 필요성 확인¶

    구분 P@3 LLM 필요 여부
    정규/크로스도메인 (A+C) 80.0% 벡터 검색만으로 충분
    구어체/슬랭 (B+D+E) 51.6% LLM Query Rewriting 필수

    500개 질의의 50% (B+D+E = 250개)가 구어체/슬랭 질의이며, 이 구간에서의 P@3가 51.6%로 전체 성능을 끌어내리는 주요 원인이다. 이는 P1-a Query Rewriting 파이프라인의 필요성을 강하게 뒷받침한다.

    우선순위 작업 해결하는 약점 기대 효과

    성과 궤적 (Phase 2 누적)¶

    Phase 2 검색 성능 개선 궤적:

Precision@3:
  Baseline (Phase 2 초기, 10개):     80%
  + Contextual Retrieval (P0-a):      90%   (+10%p)  [25개 질의]
  + BGE-M3 D+S RRF (P0-b):          92%   (+2%p)   [25개 질의]
  ── 500개 질의 확장 (2026-03-28) ──
  Setting C Baseline:                65.8%            [500개 질의]
    ├ 정규/크로스도메인 (A+C):         80.0%            ← 벡터 검색 유효
