    Step 5 — Cross-Encoder 점수 퓨전 벤치마크 결과¶

    테스트 일시: 2026-03-28 환경: DGX Spark, rag-embedding 컨테이너 (BGE-M3 + Qdrant v1.17.0) Cross-Encoder: dragonkue/bge-reranker-v2-m3-ko 점수 퓨전: α=0.5, final = 0.5 × rrf_rank_score + 0.5 × ce_score 결과 파일: results/p1b_step5_rerank.json

    1. 3-Way 비교표 (Baseline vs Full vs Full+Rerank)¶

    세트 Baseline P@3 Full P@3 (Step1~4) Full+Rerank P@3 (Step5) Full 변화 Rerank 변화 최적
    A (정규 150) 79% (119) 78% (117) 79% (119) -1%p 0%p Rerank
    B (구어체 50) 46% (23) 58% (29) 56% (28) +12%p +10%p Full
    C (교차 100) 81% (81) 81% (81) 81% (81) 0%p 0%p 동일
    D (구어체 100) 50% (50) 55% (55) 57% (57) +5%p +7%p Rerank
    E (혼합 100) 56% (56) 54% (54) 55% (55) -2%p -1%p Rerank
    전체 65.8% (329) 67.2% (336) 68.0% (340) +1.4%p +2.2%p Rerank

    2. Step 5 CE 점수 퓨전 효과 분석¶

    2-1. CE가 개선한 영역¶

    세트 Full → Rerank 분석
    Set A 78% → 79% (+1%p) CE가 REWRITE 회귀를 복원. Full에서 잘못 변환된 질의를 CE가 원래 순위로 복구
    Set D 55% → 57% (+2%p) CE가 구어체 변환 후 후보 품질을 정밀 선별. 하이브리드+CE 시너지
    Set E 54% → 55% (+1%p) 소폭 개선. 인터넷 속어 질의에서 CE가 미세 리랭킹 기여

    2-2. CE가 하락시킨 영역¶

    세트 Full → Rerank 분석
    Set B 58% → 56% (-2%p) CE 점수 양극화 문제 잔존. 극단 구어체에서 CE가 정답을 밀어냄

    2-3. 핵심 발견¶

    1. CE 점수 퓨전은 이전 Setting E(-3.2%p 회귀)와 완전히 다른 결과: 전체 +2.2%p 개선
    2. 핵심 차이: Setting E는 CE 단독 교체였으나, Step 5는 Query Rewriting 후 RRF 결과에 CE를 합산 (점수 퓨전)
    3. Set A 회귀 복원: Full에서 -1%p였던 것을 CE가 0%p로 복구 → 회귀 방지 효과
    4. Set D 최고 성능: 50% → 57% (+7%p)는 전 세트 중 가장 큰 절대 개선

    3. 이전 Setting E와의 비교¶

    지표 Setting E (CE 단독 교체) Step 5 (점수 퓨전) 차이
    전체 P@3 62.6% (-3.2%p 회귀) 68.0% (+2.2%p) +5.4%p
    Set B P@3 38% (-8%p) 56% (+10%p) +18%p
    Set D P@3 47% (-3%p) 57% (+7%p) +10%p
    방식 CE 점수로 RRF 순위 전면 교체 α=0.5 convex combination 점수 퓨전이 핵심

    → CE 점수를 RRF와 합산하는 점수 퓨전 방식이 CE 단독 교체보다 5.4%p 우수

    4. 전체 성과 궤적¶

    Phase 2 검색 성능 궤적:

── 이전 ──
  초기:                              80%    (25개)
  + Contextual Retrieval:            90%    (+10%p)
  + BGE-M3 D+S RRF:                92%    (+2%p)

── 500개 확장 ──
  Baseline (Setting C):              65.8%  (500개)
  Setting E (CE 단독):               62.6%  (-3.2%p, 미채택)

── P1-a/b Step 1~5 ──
  + Full Pipeline (Step 1~4):        67.2%  (+1.4%p)
  + Full+Rerank (Step 5, CE 퓨전):   68.0%  (+2.2%p) ← 현재 최고

  세트별:
    A (정규):     79% → 79%   (0%p, 안정)
    B (구어체):   46% → 56%   (+10%p)
    C (교차):     81% → 81%   (0%p, 안정)
    D (구어체):   50% → 57%   (+7%p)
    E (슬랭):    56% → 55%   (-1%p)

    5. 목표 대비 달성도¶

    지표 목표 Baseline Full+Rerank 달성
    Set A P@3 ≥ 85% 79% 79% 미달
    Set B P@3 ≥ 65% 46% 56% 미달 (목표 대비 -9%p)
    Set C P@3 ≥ 85% 81% 81% 미달
    Set D P@3 ≥ 60% 50% 57% 미달 (목표 대비 -3%p)
    Set E P@3 ≥ 65% 56% 55% 미달
    전체 P@3 ≥ 75% 65.8% 68.0% 미달 (목표 대비 -7%p)
    전체 P@3 ≥ 65.8% (Baseline 이상) — 68.0% 달성 (+2.2%p)

    6. 판정 — Full+Rerank를 프로덕션 채택¶

    설정 전체 P@3 구어체 평균 (B+D) 정규 평균 (A+C) 판정
    Baseline 65.8% 48.0% 80.0% 기존
    Full (Step 1~4) 67.2% 56.5% 79.5% 구어체 개선, 정규 소폭 하락
    Full+Rerank (Step 5) 68.0% 56.5% 80.0% 최고 P@3 + 정규 보호

    Full+Rerank 채택 근거: 1. 전체 P@3 68.0% — 3개 설정 중 최고 2. 정규 질의 (A+C) 80.0% — Baseline과 동일, 회귀 없음 3. 구어체 (B+D) 56.5% — Baseline 48.0%에서 +8.5%p 4. CE가 Set A 회귀를 복원하고 Set D에서 추가 +2%p
