    P1-a Query Decomposition 벤치마크 결과 (500개 질의 확장 Baseline)¶

    테스트 일시: 2026-03-28 환경: DGX Spark, rag-embedding 컨테이너 (BGE-M3 + Qdrant v1.17.0) LLM: Claude Code CLI (claude-code -p --model sonnet) 테스트 스크립트: codes/embedding/search_test_phase2_v2.py + codes/query/test_queries.py 질의 소스: data/test_question_v0.md — 500개 (A:150, B:50, C:100, D:100, E:100) 상태: Baseline 결과만 수록 — Full LLM Pipeline 벤치마크는 미실행

    1. 종합 비교표 — 500개 질의 Baseline¶

    세트 질의 수 Baseline P@3 정답/총 Avg Top-1 p95
    A (정규 질의) 150 79.3% 119/150 0.8294 61ms
    B (극단 구어체) 50 46.0% 23/50 0.7096 62ms
    C (크로스 도메인) 100 81.0% 81/100 0.7942 62ms
    D (극단 구어체) 100 50.0% 50/100 0.7270 64ms
    E (혼합형) 100 56.0% 56/100 0.7824 65ms
    전체 500 65.8% 329/500 0.7885 65ms

    Setting C (Dense+Sparse RRF) 기준. Full LLM Pipeline (Baseline vs Full) 비교는 별도 실행 필요.

    2. 세트별 분석¶

    2-1. Set A — 정규 질의 (150개)¶

    지표 값 분석
    P@3 79.3% (119/150) 이전 25개(92%) 대비 하락. A26~A150에 난이도가 높은 질의가 포함됨
    Avg Top-1 0.8294 전 세트 중 가장 높은 Top-1 스코어
    특징 전문/반전문 용어 포함 Dense+Sparse RRF가 안정적으로 작동하는 구간

    2-2. Set B — 극단 구어체 (50개)¶

    지표 값 분석
    P@3 46.0% (23/50) 절반 이상 실패 — LLM 없이는 해결 불가
    Avg Top-1 0.7096 전 세트 중 가장 낮은 Top-1 스코어
    특징 전문용어 부재, 일상어 질의 "나라에 돈", "몇 프로까지" 등 어휘 격차 극대화

    2-3. Set C — 크로스 도메인 (100개)¶

    지표 값 분석
    P@3 81.0% (81/100) 전 세트 중 최고 P@3
    Avg Top-1 0.7942 양호
    특징 정규 용어 + 도메인 교차 "재건축+양도세", "청약+대출" 등 정규 용어가 있으면 도메인 교차에도 검색 성공

    2-4. Set D — 극단 구어체 (100개)¶

    지표 값 분석
    P@3 50.0% (50/100) Set B와 유사한 패턴
    Avg Top-1 0.7270 Set B보다 소폭 높음
    특징 일상어, 구어체 표현 구어체 ↔ 전문용어 사이의 어휘 격차가 핵심 병목

    2-5. Set E — 혼합형 (100개)¶

    지표 값 분석
    P@3 56.0% (56/100) 인터넷 슬랭이 성능 저하의 주 원인
    Avg Top-1 0.7824 Set A, C보다는 낮지만 B, D보다 높음
    특징 정규+슬랭 혼합 인터넷 슬랭("영끌", "갭투" 등)이 임베딩 공간에서 전문용어와 거리가 멈

    3. Set B 집중 분석 — 46% Baseline이 확인하는 LLM 필요성¶

    3-1. Set B의 핵심 문제¶

    Set B(극단 구어체, 50개)의 P@3 46.0%는 LLM Query Rewriting 없이는 절반 이상의 구어체 질의를 해결할 수 없음을 확인한다.

    유형 예시 질의 기대 결과 실패 원인
    어휘 격차 "나라에 돈 내야 되나" 취득세 "나라에 돈" ↔ "취득세" 의미적 거리
    비유적 표현 "몇 프로까지 빌려주는지" LTV/담보인정비율 "프로" ↔ "비율", "빌려주는" ↔ "대출한도" 격차
    슬랭 "영끌해도 되나" 대출한도/리스크 인터넷 슬랭이 임베딩 공간에 부재
    생략 표현 "세금 얼마야" 취득세/양도세 맥락 없는 축약 표현

    3-2. 이전 42개 벤치마크에서의 LLM 효과 (참고)¶

    이전 42개 질의 벤치마크(Phase 2 Legacy, 아래 섹션 7 참조)에서 Set B(2개 실패 질의)가 LLM Query Rewriting으로 100% 해결된 바 있다. 500개 확장에서도 Set B의 나머지 질의들에 대해 동일한 효과가 기대된다.

    4. Set C 분석 — 정규 용어가 있으면 크로스 도메인도 성공¶

    4-1. Set C의 강점¶

    Set C(크로스 도메인, 100개)의 P@3 81.0%는 전 세트 중 최고이다. 정규 용어가 포함되어 있으면 두 도메인을 교차하는 질의에서도 Dense+Sparse RRF가 효과적으로 작동한다.

    질의 유형 예시 P@3 경향
    정규+정규 교차 "재건축 아파트 양도세" 높음 — 양 도메인 키워드 매칭
    정규+구어체 교차 "경매 집 사면 세금 얼마야" 보통 — 한쪽 키워드로 부분 매칭
    구어체+구어체 교차 "집 팔고 남은 돈에서 떼가나" 낮음 — Set B/D와 동일 문제

    4-2. 시사점¶

    크로스 도메인 질의의 핵심 성공 요인은 도메인 교차 자체가 아니라 정규 용어의 존재 여부이다. Set C의 81%는 대부분 정규 용어가 포함된 질의에서 달성된 것이며, 구어체만으로 구성된 크로스 도메인 질의는 Set B/D와 유사한 패턴으로 실패한다.

    5. 500개 질의 Baseline에서의 핵심 발견¶

    5-1. Baseline P@3 하락 분석¶

    비교 P@3 분석
    이전 25개 질의 (Set A 기존) 92% 정규 질의 위주로 구성
    500개 전체 65.8% 극단 구어체(B+D)와 슬랭(E) 포함으로 하락
    500개 중 정규+크로스 (A+C) 80.0% 이전 25개와 유사한 난이도 질의는 여전히 양호
    500개 중 구어체+슬랭 (B+D+E) 51.6% 핵심 병목 구간

    5-2. 세트별 핵심 수치¶

    발견 세트 P@3 의미
    Set A baseline 하락 A 79.3% 이전 92% 대비 -12.7%p. A26~A150에 난이도 높은 질의 포함
    크로스 도메인 양호 C 81.0% 정규 용어가 있으면 도메인 교차에도 검색 성공
    극단 구어체 병목 B+D ~48% P1-a Query Rewriting의 핵심 타겟
    혼합형 중간 E 56.0% 슬랭 질의가 성능 저하의 주 원인

    5-3. P1-a Query Rewriting 필요성 재확인¶

    500개 질의에서 구어체/슬랭 질의(B+D+E = 250개)의 Baseline P@3가 51.6%로 확인되었다. 이는: - 전체 500개 질의의 절반이 구어체/슬랭으로 구성 - 해당 구간에서 절반 이상이 실패 - 세 가지 검색 설정(C, E, F) 모두에서 공통적으로 실패

    → 검색 설정 변경으로는 해결 불가, LLM Query Rewriting이 필수임을 500개 규모에서 재확인.

    6. 업데이트된 로드맵 — P1-a-1 ~ P1-b 순차 개선¶

    우선순위 작업 해결하는 약점 기대 효과 소요
    P1-a-1 하이브리드 전략 (원본+변환 동시 검색) 과잉 변환 회귀 정규 질의 P@3 유지 + 구어체 개선 0.5일

    P1-b Cross-Encoder 재도입 정밀 리랭킹 RRF 후보 풀 개선 후 CE 효과 재검증 1일
