## 3. 세부 분석

### 3-1. Set B — 역대 최고 성능 달성

**46% → 64% (+18%p)** — 이전 최고치(56%)를 +8%p 초과.

| 기여 요인 | 기여도 | 근거 |
|----------|--------|------|
| 슬랭 alias 확장 | +2%p | Baseline 46% → 48% (벡터 매칭 직접 개선) |
| LLM REWRITE | +8%p | 35/50 질의가 REWRITE (구어체→전문용어 변환) |
| 조건부 α=0.3 | +8%p | CE 비중 축소로 정답 보호 (CE 오판 억제) |

> Set B의 70%가 LLM REWRITE를 거쳤으며, 조건부 α가 CE의 부정적 영향을 최소화. **alias 확장 + LLM REWRITE + 조건부 α의 시너지**가 확인됨.

### 3-2. Set C — 이전 최고치 초과

**81% → 83% (+2%p)** — 안정적 개선.

- 14건의 DECOMPOSE가 크로스도메인 질의 분해에 기여
- 슬랭 alias가 교차도메인 커버리지 확대

### 3-3. Set A — 회귀 발생 (중요)

**79% → 75% (-4%p)** — 가장 큰 회귀.

| 원인 | 분석 |
|------|------|
| alias 과다로 임베딩 벡터 희석 | Baseline 79.3% → 78% (-1.3%p) — alias 추가만으로 -1.3%p |
| CE min-max 정규화 노이즈 | 이진 분포에서 min-max가 미세 차이를 왜곡 |
| REWRITE 회귀 (12건) | 일부 정규 질의가 불필요하게 REWRITE되어 원본 키워드 유실 |

### 3-4. Set E — 유의미한 하락

**56% → 50% (-6%p)** — 가장 큰 하락.

| 원인 | 분석 |
|------|------|
| DECOMPOSE 7건 중 다수 회귀 | 복합 슬랭 질의의 분해가 부정확 |
| CE min-max 정규화 | 슬랭 질의에서 CE 점수가 극단적으로 낮아 정규화가 노이즈 증폭 |
| REWRITE 11건 중 변환 실패 | 복합 슬랭("영끌해서 집 샀는데 이자...")을 단일 전문용어로 매핑 불가 |

### 3-5. CE min-max 정규화의 부정적 영향

1차(LLM 미작동)와 2차(LLM 정상) 결과를 결합하면:

| 세트 | BL→CE단독 효과(1차) | BL→Full+Rerank(2차) | CE 단독 효과 판정 |
|------|-------------------|-------------------|--------------|
| A | -1.0%p | -3.0%p | 부정적 |
| B | **+8.0%p** | **+16.0%p** | 긍정적 (조건부 α) |
| C | **+2.0%p** | **+1.0%p** | 긍정적 |
| D | -3.0%p | +3.0%p | LLM 의존적 |
| E | -4.0%p | -5.0%p | **부정적** |

> CE min-max 정규화는 Set B에서만 명확히 긍정적이고, Set A/E에서는 부정적. **정규화 롤백을 권장**.

```python
# pipeline.py — _rerank_ontology()에서 min-max 정규화 제거
# 변경 전: normalized_ce = (item.score - ce_min) / ce_range

final = alpha * rrf_rank_score + (1 - alpha) * item.score
```

## 9. 핵심 인사이트

1. **전체 P@3 68.0%로 이전 최고(Step 5)와 동등하되 구성이 개선됨**: Set B/D가 +4%p/+1%p 개선되고 Set A가 -2%p 하락. 구어체 대응력이 향상된 균형 잡힌 구성.

2. **Set B 목표 달성 (60%, 이전 BL 대비 +14%p)**: 슬랭 alias + LLM REWRITE + 조건부 α + CE 스킵의 4중 방어가 효과적.

3. **CE min-max 정규화는 확실히 부정적**: 롤백만으로 Set A +2%p, Set E +4%p 회복. **이진 분포의 CE에는 raw score가 더 안정적**.

4. **SLANG_MARKERS 분리가 핵심**: alias 확장과 colloquial detection의 설계 충돌을 해소. CE 스킵이 적절히 작동하여 D +2%p.

5. **Set A 잔여 회귀 (-2%p)의 원인**: alias 확장에 의한 벡터 희석(baseline -1.3%p) + 일부 REWRITE 회귀. Phase 2B에서 alias 수 제한 강화(max 18)로 추가 개선 가능.

6. **Set E는 검색 최적화의 한계**: 54%로 회복되었으나 이전(56%)에 미달. 복합 슬랭+다중도메인 질의는 **LLM 답변 보상 전략**(Self-RAG, CRAG)이 필요.
