## 4. TEST 3: chunk_type별 검색 성능

질의: "취득세 세율 6억원 9억원"

| chunk_type | Top-1 Score | 특성 |
|-----------|------------|------|
| prose | 0.591 | 해당 섹션 원문 매칭 |
| table_fact | 0.500 | 과세표준·세율 표 행 매칭 |
| **hyde** | **0.833** | 구어체 질문 매칭 — 최고 점수 |

**분석**: HyDE 질문이 사용자 구어체 질의와 가장 유사하므로 Top-1 score가 가장 높음. 실제 RAG 파이프라인에서는 hyde로 매칭 후 동일 chunk_id의 prose/table_fact를 함께 제공하는 2단계 전략이 유효.

### 개선 가능 영역

1. **cross-domain 질의** ("경매 낙찰 + 세금"처럼 두 브랜치에 걸치는 질의)
   - 현재 단일 벡터 매칭으로는 한 브랜치에 편향됨
   - Phase 1 RAG 파이프라인에서 Query Rewriting + Multi-hop 검색으로 보완 가능

2. **"집 살 때 세금"류 추상적 질의**
   - 일반적인 "세금" 관련 용어가 상위로 올라가고 "취득세"가 4위에 위치
   - Query Rewriting 레이어에서 "집 살 때 → 취득세"로 정규화하면 해결

3. **legal_docs table_fact 정밀도**
   - row_fact 텍스트가 context 부족 (행 단위 분리로 인한 정보 손실)
   - 계층경로 prefix가 도움이 되나, 표 전체 맥락이 필요한 경우 prose와 cross-reference 필요
