## 2. 설정 E (Cross-Encoder) 상세 분석

### 2-1. 성공 사례 — 기존 실패 질의 해결

Cross-Encoder가 기존 RRF에서 해결하지 못한 질의를 해결한 사례:

| 질의 | Setting C 결과 | Setting E 결과 | 변화 |
|------|---------------|---------------|------|
| "집 살 때 세금 얼마야" | 과세표준 (Top-1) | **취득세 (Top-1, 0.9346)** | **최초 정답 Top-1 달성!** |

이 질의는 Phase 2 전체를 통틀어 처음으로 "취득세"가 Top-1에 올라온 경우이다. Cross-Encoder의 깊은 맥락 이해 능력("집 살 때 세금" ↔ "취득세")이 효과적으로 작동했다.

법률 질의에서도 우수한 성과:
- "취득세 6억원 초과 세율" → **0.9999** (거의 완벽 매칭)
- "1세대 1주택 양도세 비과세" → **0.9999**
- "종합부동산세 과세기준일" → **0.9999**

### 2-2. 실패 사례 — P@3 하락 원인

| 질의 | Setting C 결과 (P@3) | Setting E 결과 (P@3) | 실패 원인 |
|------|---------------------|---------------------|----------|
| "경매 낙찰되면 세금 내야해?" | O (낙찰 Top-3) | **X** (법정매각조건, 인수주의, 교부청구) | CE가 "경매" 관련 법률용어에 높은 점수를 주지만 "세금" 키워드 미포함 |
| "전세사기 예방 방법" | O (보험 Top-3) | **X** (허위계약, 위장거래, 부재지주) | CE가 "사기"에 집중하여 "예방/보호" 맥락을 놓침 |
| "월급 대비 빚 갚는 비율" | O (부채서비스액 Top-3) | **X** (완성도기준지급, 기준경비율, 세율) | CE가 "비율/기준" 키워드에 잘못된 항목을 매칭 |
| "은행에서 몇 프로까지" | X (공통 실패) | **X** | CE도 LTV/담보 연결 실패 |
| "부동산 사면 나라에 돈" | X (공통 실패) | **X** | CE도 취득세 연결 실패 (국세를 Top-1으로 선택) |
| "월세 올려달라는데 한도" | O (차임증감청구권) | **X** | CE가 "임차료"에 높은 점수, "상한/인상/증액" 미매칭 |

### 2-3. Cross-Encoder의 특성

| 관찰 | 설명 |
|------|------|
| **이진적 스코어 분포** | 대부분의 스코어가 0에 가깝거나 1에 가까움 (0.0000 ~ 0.9999). 중간 점수가 드물다 |
| **깊은 매칭에 강함** | "집 살 때 세금"→"취득세"처럼 표면이 다른 의미 연결에 탁월 |
| **넓은 키워드 매칭에 약함** | RRF의 "여러 키워드 중 하나만 걸려도 OK" 방식보다 단일 최적 매칭에 집중 |
| **레이턴시 높음** | 50개 후보 리랭킹에 ~450ms (모델 로드 제외) |

### 2-4. 판정

Cross-Encoder는 **특정 질의에서 획기적인 개선**을 보이지만, 전체 P@3를 **16%p 하락**시킨다. 현 단계에서는 프로덕션 채택이 어렵다. 다만 향후 **Query Rewriting과 결합** 시 RRF 후보 품질이 개선되면 Cross-Encoder의 정밀 리랭킹이 더 효과적일 수 있다.

## 3. 설정 F (3-Way RRF) 상세 분석

### 3-1. Setting C 대비 변화

| 질의 | Setting C (P@3) | Setting F (P@3) | Top-3 변화 |
|------|----------------|----------------|-----------|
| "전세사기 예방 방법" | O | **O** (주택임대차신용보험 유지) | 약간 순서 변화, 정답 유지 |
| "경매 낙찰되면 세금" | O (낙찰 Top-3) | **O** (배당, 배분계산서, 낙찰) | ColBERT 투표가 "낙찰"을 유지 |
| "집 살 때 세금 얼마야" | O (취득세 Top-3) | **O** (과세표준, 취득세, 세율) | 순서 약간 변동, 정답 유지 |

### 3-2. 신규 실패

| 질의 | Setting C 결과 | Setting F 결과 | 원인 |
|------|---------------|---------------|------|
| "부동산 사면 나라에 돈" | X (공통) | X (공통) | 동일 실패 |
| "은행에서 몇 프로까지" | X (공통) | X (공통) | 동일 실패 |
| "월세 올려달라는데 한도" | **O** (차임증감청구권 Top-2) | **X** (임차료, 차임증감청구권, 갱신) | ColBERT 투표가 "임차료"를 Top-1으로 끌어올려 Top-3 구성이 변경됨. 기대 키워드 "증액/인상/상한"이 Top-3에서 빠짐 |

### 3-3. 판정

3-Way RRF는 Setting C(92%)보다 **4%p 낮은 88%**로, 근접하지만 미달이다. ColBERT가 RRF의 3번째 voter로 참여하면서 일부 질의에서 미세한 순위 변동이 발생하고, 이것이 P@3에 영향을 준다.

**장점**: 레이턴시가 128ms로 Setting C(48ms) 대비 80ms만 증가하여 서비스 수준 유지 가능.

## 4. 레이턴시 비교

| 설정 | p50 | p95 | 비고 |
|------|-----|-----|------|
| C (D+S RRF) | 8.3ms | 48ms | 기준선 |
| E (+ Cross-Encoder) | 513ms | ~530ms* | 모델 로드 후 안정 |
| F (3-Way RRF) | 66ms | 128ms | 서비스 수준 유지 |

> *Setting E의 p95가 15,589ms인 것은 첫 질의에서의 모델 로딩(15초) 때문. 모델 로드 후 순수 리랭킹 레이턴시는 ~450-530ms 수준.

## 5. 결론 및 다음 단계

### 5-1. 핵심 발견

1. **Setting C (D+S RRF)가 여전히 최적**: P@3 92%, 레이턴시 48ms. 현행 유지.

2. **Cross-Encoder(E)는 정밀도와 범위의 트레이드오프**: "집 살 때 세금"→"취득세" 최초 달성은 의미 있지만, 넓은 키워드 매칭이 약해 전체 P@3이 76%로 하락. RRF의 "다수결" 방식이 현재 데이터셋에서 더 강건하다.

3. **3-Way RRF(F)는 ColBERT 활용의 가장 안전한 방식**: P@3 88%로 Setting C에 근접. ColBERT가 voter로 참여하면서도 전체를 망치지 않는다.

### 5-2. 권장 사항

| 우선순위 | 작업 | 근거 |
|---------|------|------|
| **즉시** | Setting C 유지 (D+S RRF, P@3=92%) | 여전히 최고 성능 |
| **P1-a** | Query Rewriting (LLM 질의 변환) | 실패 질의 2개("나라에 돈", "몇 프로까지")가 의미 격차 문제 — 질의 자체를 변환해야 해결 가능 |
| **P1-b** | Cross-Encoder를 Query Rewriting 후 2차 리랭커로 도입 | 질의 품질이 개선되면 CE의 정밀 매칭이 효과적 |
| **참고** | 3-Way RRF는 ColBERT 활용이 필요한 경우 fallback으로 보존 | P@3 88%로 합리적 성능 |

### 5-3. 성과 궤적 (Phase 2 누적)

```
Phase 2 검색 성능 개선 궤적:

Precision@3:
  Baseline (Phase 2 초기):       80%
  + Contextual Retrieval (P0-a):  90%   (+10%p)
  + BGE-M3 D+S RRF (P0-b):      92%   (+2%p)   ← 현재 운영
  + Cross-Encoder (E):           76%   (-16%p)  ← 하락 (미채택)
  + 3-Way RRF (F):              88%   (-4%p)   ← 근접 (ColBERT fallback)

결론: Setting C 유지, 다음 단계는 Query Rewriting (P1-a)
```
