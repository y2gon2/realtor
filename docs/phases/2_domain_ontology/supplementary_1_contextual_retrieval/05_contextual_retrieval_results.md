## 4. 핵심 발견

### 4-1. 순수 벡터 검색은 정규/준정규 질의에서 효과적

- **Set A (79.3%)**, **Set C (81.0%)**: 정규 용어가 포함된 질의에서 Dense+Sparse RRF가 안정적으로 작동
- Avg Top-1 Score도 0.79~0.83 범위로 양호

### 4-2. 구어체 질의가 핵심 병목

- **Set B (46%)**, **Set D (50%)**: 전문용어 없이 일상어로만 구성된 질의에서 절반 이상 실패
- "나라에 돈 내야 되나" ↔ "취득세", "몇 프로까지 빌려주는지" ↔ "LTV" 사이의 의미적 거리를 벡터만으로 극복 불가

### 4-3. LLM Query Rewriting 필요성 확인

| 구분 | P@3 | LLM 필요 여부 |
|------|-----|-------------|
| 정규/크로스도메인 (A+C) | 80.0% | 벡터 검색만으로 충분 |
| 구어체/슬랭 (B+D+E) | 51.6% | **LLM Query Rewriting 필수** |

500개 질의의 50% (B+D+E = 250개)가 구어체/슬랭 질의이며, 이 구간에서의 P@3가 51.6%로 전체 성능을 끌어내리는 주요 원인이다. 이는 **P1-a Query Rewriting 파이프라인**의 필요성을 강하게 뒷받침한다.
