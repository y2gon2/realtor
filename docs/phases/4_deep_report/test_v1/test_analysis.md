## 3. 테스트 케이스별 결과

### 3.1 테스트 1: 왕십리 텐즈힐 (매매)

- **입력**: `왕십리 텐즈힐`
- **목적**: 매매_실거주
- **소요 시간**: 195.6초
- **데이터 완전성**: 57.1% (7개 중 4개 수집)

#### 주소 정규화

| 항목 | v1.0 (4/5) | v1.1 (4/6) |
|------|-----------|-----------|
| 단지명 | ❌ 왕십리텐즈힐공인중개사 | ✅ **텐즈힐1단지아파트** |
| 도로명 | 서울 성동구 왕십리로 362 | **서울 성동구 무학로 33** |
| 지번 | 서울 성동구 도선동 363-1 | **서울 성동구 하왕십리동 1066** |
| 시군구코드 | 11200 | 11200 |
| PNU | 1120010400203630001 | **1120010200210660000** |

→ 주소 오매칭 문제 **해결됨**. 공인중개사가 아닌 실제 아파트 단지로 매칭.

#### API 데이터 수집

| API | 상태 | 상세 |
|-----|:----:|------|
| 매매 실거래가 | ✅ | **0건** — 최근 12개월 해당 단지 매매 거래 없음 |
| 전월세 실거래가 | ✅ | **0건** — 전월세 거래도 없음 |
| 건축물대장 | ✅ | 텐즈힐, **1,702세대**, 층수 누락, 건축년도 누락 |
| 토지이용규제 | ❌ | 서버 500 (승인 반영 대기) |
| 주변시설 (입지) | ✅ | 종합 **77점**, 상왕십리역 2호선 (504m) |

#### 분석

**거래 0건 원인**: 텐즈힐 1단지의 실거래가 API 단지명이 코드에서 사용하는 필터명과 불일치할 가능성. 실거래가 API에서는 `텐즈힐`이 아닌 `텐즈힐1단지` 또는 `텐즈힐 1단지` 등 다른 형태일 수 있음. 현재 부분 일치(`in`) 검색이지만 카카오에서 반환한 `텐즈힐1단지아파트`와 실거래가 API의 `aptNm` 필드 간 매칭 실패.

**건축물대장 층수/건축년도 누락**: 총괄표제부(getBrRecapTitleInfo)에서 `grndFlrCnt`, `useAprDay` 필드가 빈 값으로 반환됨. 표제부(getBrTitleInfo) 병행 호출로 보완 필요.

**입지 점수 77점**: v1.0에서 92점(공인중개사 위치)이었으나, 실제 아파트 위치로 수정되어 77점으로 변경. 상왕십리역 504m (도보 8분).

#### LLM 섹션 생성

| 섹션 | 상태 | 글자 수 | 비고 |
|------|:----:|:------:|------|
| 가격/시세 분석 | ✅ | 1,827 | 데이터 부족 명시, 대안 제시 |
| 입지 분석 | ✅ | 2,911 | 상왕십리역 도보 8분, 주변 시설 분석 |
| 법률/규제 분석 | ✅ | 2,726 | 세대수(1,702) 기반 분석 |
| 투자 수익률 | ✅ | 1,012 | 가격 데이터 없어 시뮬 불가 명시 |
| 요약 결론 | ✅ | 616 | |
| 차트 | 없음 | - | 거래 0건으로 생성 불가 |
| 룰 엔진 | ❌ | - | 매매 중앙값 없어 실행 불가 |
