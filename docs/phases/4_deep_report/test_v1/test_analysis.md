### 3.3 발견된 문제점 (상세)

#### A. 주소 정규화 문제

| 문제 | 테스트 | 상세 | 심각도 |
|------|--------|------|:------:|
| **단지명 오인식** | 왕십리텐즈힐 | 카카오 키워드 검색 결과 1위가 "왕십리텐즈힐**공인중개사**" (부동산 중개사무소)로 반환됨. 아파트 단지가 아닌 중개사무소가 매칭되어 전체 분석이 잘못된 대상으로 수행됨. | **높음** |
| **건축물대장 실패** | 왕십리텐즈힐 | 잘못된 PNU(중개사무소)로 조회하여 건축물대장 미수집. 실제 텐즈힐 아파트와 전혀 다른 주소. | 높음 |
| **거래 0건** | 왕십리텐즈힐 | 잘못된 시군구코드(도선동)로 조회하여 텐즈힐 아파트 거래를 못 찾음. 실제 텐즈힐은 하왕십리동 소재. | 높음 |
| **동/호 파싱 정상** | 반포자이 | "1012동 5층" → 동=1012, 호=None 정상 분리. 검색어 "반포자이"로 정리됨. | 정상 |

**근본 원인**: 카카오 키워드 검색이 아파트 단지가 아닌 주변 상호(공인중개사 등)를 1순위로 반환하는 경우가 있음. 해결 방안:
1. 검색 결과에서 카테고리가 "부동산 > 주거시설 > 아파트"인 것만 필터
2. 검색어에 "아파트"를 자동 추가하여 재검색
3. 후보 다중 표시 후 사용자가 선택 (인터뷰 UI 활용)

#### B. API 데이터 수집 문제

| 문제 | 테스트 | 상세 | 원인 |
|------|--------|------|------|
| **전월세 API 403** | 양쪽 모두 | data.go.kr 서버 호출 시 403 Forbidden | API 활용 신청 승인 후 서버 반영 지연 (수 시간~1일) |
| **토지이용규제 500** | 양쪽 모두 | data.go.kr 서버 호출 시 500 Server Error | 동일 원인 (승인 반영 지연) |
| **건축물대장 층수 누락** | 반포자이 | `floors: None` — 총괄표제부 API에서 `grndFlrCnt` 미반환 | 총괄표제부에서 층수가 빠지는 경우 존재. 표제부(getBrTitleInfo) 병행 호출 필요 |
| **전월세 0건** | 왕십리텐즈힐 | 전월세 거래 데이터 0건 | 주소 오인식(중개사무소) + API 미반영 이중 원인 |

#### E. 성능

| 항목 | 왕십리텐즈힐 | 반포자이 |
|------|:-----------:|:--------:|
| 총 소요 시간 | 178.6초 | 266.3초 |
| API 수집 | 7.2초 | 7.2초 |
| LLM 생성 (4섹션 + 요약) | **171초** | **259초** |

**병목: LLM CLI 호출.** API 수집은 7초로 빠르지만, CLI subprocess 호출이 전체 시간의 95%+ 차지. 각 섹션을 순차 호출하므로 5회 × 30~50초 = 150~250초. API 모드(SDK) 전환 시 병렬 호출로 60초 이내 가능.

### 3.4 추가 필요 사항 (우선순위 순)

#### 우선순위 높음 (보고서 품질에 직접 영향)

1. **주소 검색 정확도 개선** — 카카오 키워드 검색 시 아파트 카테고리 필터 적용. 검색어에 "아파트" 자동 추가. 후보 다중 표시 후 사용자 선택.
2. **전월세/토지이용규제 API 승인 반영 대기** — 서버 반영 후 자동 연동됨 (코드 변경 불필요).
3. **목적별 프롬프트 분기** — "전세" 목적 시 전세 분석 위주, "매매" 시 매매 위주로 섹션 프롬프트 조정.
4. **건축물대장 표제부 병행 호출** — 총괄표제부에서 누락되는 층수/구조 등을 표제부(getBrTitleInfo)로 보완.

#### 우선순위 중간 (기능 확장)

5. **한국어 폰트 수정** — matplotlib에서 .ttc 파일을 폰트 인덱스 지정으로 로드 (`fontIndex=1`).
6. **일조/조망 분석** — VWorld WFS + suncalc 연동.
7. **리스크 분석** — KOSIS 인구통계 + 공급 전망 연동.
8. **미래 가치** — 뉴스 API(네이버/빅카인즈) 연동.

#### 우선순위 낮음 (성능/UX 개선)

9. **LLM API 모드 전환** — CLI → SDK로 전환하여 병렬 호출, 60초 이내 목표.
10. **Chainlit UI 브라우저 테스트** — 인터뷰 플로우 실 사용 테스트.
11. **차트 한글 깨짐 외 미세 품질** — Y축 단위, 범례 위치, 색상 최적화.
