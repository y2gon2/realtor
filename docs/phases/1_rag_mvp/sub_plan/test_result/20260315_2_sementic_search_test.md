## 5. 이전 테스트 대비 변화 분석

### 5.1 핵심 개선: Hybrid 검색 활성화

| 항목 | 테스트 #1 (Sparse 비활성) | **테스트 #2 (BM25 활성)** |
|---|---|---|
| Hybrid 동작 여부 | 비활성 (빈 SparseVector) | **활성** (Kiwi 형태소 분석 기반) |
| Dense-Hybrid 일치율 | 100% (동일 결과) | **52%** (상호보완) |
| Hybrid 고유 결과 비율 | 0% | **~48%** |
| Hybrid top-1 score 분포 | 모두 0.5000 고정 | 0.5000~0.6111 차별화 |

### 5.2 Hybrid가 Dense 대비 추가로 잡아낸 문서 유형

Hybrid가 Dense에 없던 결과를 반환한 케이스를 분석하면:

| 유형 | 빈도 | 예시 |
|---|---|---|
| **키워드 정확 매칭** 기반 실무 콘텐츠 | 높음 | #4: "DSR" 약어가 제목에 포함된 대출 용어 설명 |
| **정책 관련** 최신 문서 | 중간 | #15: 이재명 정부 부동산 정책 방향 |
| **실무 Q&A** (경매 절차, 입찰 방식 등) | 중간 | #11: 대리입찰, 입찰액 기재 방식 |
| **BM25 노이즈** (키워드 일치하지만 맥락 불일치) | 낮음 | #16: "아파트 단지" 매칭으로 서울 단지가 인천 질의에 혼입 |

### 5.3 Dense vs Hybrid 역할 정리

| 검색 모드 | 강점 | 약점 |
|---|---|---|
| **Dense** | 질의 의도를 의미적으로 파악, 동의어/간접 표현에 강함 | 고유 키워드(약어, 고유명사) 매칭이 간접적 |
| **Hybrid** | 정확한 키워드 매칭, 약어·고유명사에 강함 | 의미적 맥락 없이 키워드만 일치하는 노이즈 가능 |
