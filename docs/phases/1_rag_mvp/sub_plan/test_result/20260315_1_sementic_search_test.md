## 5. 분석 및 평가

### 5.1 Dense 검색 품질 — 우수

- 전체 15개 질의에 대해 Dense top-1 score **평균 0.76** 수준으로, 질의 의도와 부합하는 문서를 상위에 안정적으로 반환하고 있다.
- 특히 명확한 키워드가 포함된 질의(#4 DSR: 0.8163, #7 양도세: 0.8197, #8 청약: 0.8412, #10 내 집 마련: 0.8496)에서 매우 높은 점수를 기록했다.
- 간접적 표현 질의(#9 "집값이 오를 곳": 0.7178, #14 "좋은 중학교": 0.7092)에서도 의미적으로 관련된 문서를 잘 검색하여 KURE-v1 모델의 한국어 의미 검색 능력이 양호함을 확인했다.

### 5.2 Hybrid 검색 — Sparse(BM25) 인덱스 점검 필요

- **모든 Hybrid top-1 score가 0.5000으로 고정**되어 있다. 이는 RRF 융합 점수가 정상적으로 계산되지 않거나, Sparse 벡터가 비어 있어 Dense 결과만으로 RRF가 산출되고 있을 가능성이 높다.
- 실제 `search_hybrid()` 코드에서 `SparseVector(indices=[], values=[])`로 빈 스파스 벡터를 전달하고 있어 **BM25 검색이 사실상 비활성화** 상태이다.
- Dense-Hybrid 일치율이 거의 100%인 것도 이를 뒷받침한다 — Hybrid가 Dense와 동일한 결과를 반환하므로 차별적 가치가 없는 상태.

> **조치 필요:** Sparse 인덱스에 실제 BM25 토큰을 업서트하거나, 쿼리 시점에 질의 텍스트로부터 Sparse 벡터를 생성하여 전달해야 진정한 Hybrid 검색이 동작한다.

### 5.3 chunk_type별 성능 — hyde > atomic_fact > summary

- **hyde 청크가 가장 높은 검색 점수**(평균 0.7278)를 기록했다. HyDE(Hypothetical Document Embedding) 방식으로 생성된 질문형 청크가 사용자의 자연어 질의와 의미적으로 가장 잘 매칭된다.
- atomic_fact(0.7088)도 준수하며, 구체적 사실 기반 질의에 강점을 보인다.
- summary(0.6596)는 상대적으로 낮지만, 전체적인 맥락 파악에 유용할 수 있다.

> **RAG 파이프라인 권장:** 검색 시 hyde 청크를 우선 검색하되, 응답 생성 시에는 해당 doc_id의 summary/atomic_fact를 함께 컨텍스트로 제공하는 2단계 전략이 효과적일 수 있다.

### 5.4 메타데이터 필터링 — 정상 동작

- topic_tag, region_tag, chunk_type 단일 필터 및 복합 필터 모두 정상 동작한다.
- 필터 적용 시 검색 범위가 좁아져 score가 다소 낮아지는 것은 예상된 동작이다.

### 5.5 검색 속도 — 양호

- 모델 초기 로드(cold start) 이후 Dense/Hybrid 모두 **~50ms** 이내로 응답하여 실시간 서비스에 충분한 성능이다.
- 93,943 포인트 규모에서 이 속도는 Qdrant의 HNSW 인덱스가 정상 동작하고 있음을 의미한다.
