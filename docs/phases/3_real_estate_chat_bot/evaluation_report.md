# RAG+LLM vs 순수 LLM 답변 품질 비교 평가 보고서

> 작성일: 2026-04-04 (최종 업데이트: 2026-04-05)
> 선행 문서: `00_overview.md`, `result/summary.md`, `result_v3/summary.md`, `opus_result/summary.md`
> 목적: 동일 197개 부동산 질문에 대한 RAG+LLM(Sonnet) vs 순수 LLM(Opus 4.6) 답변의 체계적 비교 평가

## 1. Executive Summary

### 핵심 수치 (최종: realestate_v3 연동 후)

| 지표 | RAG+LLM (v3 연동) | 순수 LLM (Opus 4.6) | 비고 |
|------|-------------------|---------------------|------|
| 총 질문 수 | 197개 | 197개 (+ 564 후속) | |
| **문서평가 pass** | **128개 (65.0%)** | N/A | v3 미연동 시 21개 (10.7%) |
| 할루시네이션 pass | 167개 (84.8%) | 미평가 | v3 미연동 시 166개 (84.3%) |
| 평균 소요시간 | 63,652ms (~64초) | 35,717ms (~36초) | v3 미연동 시 91초 |
| 평균 LLM 호출 | ~3.3회/질문 | 1회/질문 | |
| 검색 포인트 | **185,365** | N/A | v3 미연동 시 2,921 |
| realestate_v3 출처 | **408건** | N/A | YouTube 전문가 콘텐츠 |
| 후속 제안 수락 | N/A | 69건 (35%) | |

### 핵심 발견 3가지

**1. realestate_v3 연동으로 doc pass율 10.7% → 65.0% 급등 (+54.3%p)**
- 검색 포인트 63배 확대 (2,921 → 185,365)
- YouTube 전문가 콘텐츠가 시황/지역/실전/정책/생활 밀착형 질문을 커버
- Part A 73%, Part C 58%, Part D 57% — 모든 Part에서 균등한 개선

**2. RAG의 강점은 "출처 인용 + 광범위 커버리지"로 확대**
- 세금/법률: 법률 조문 인용 기반 정밀 답변 (기존 강점 유지)
- 시황/투자: YouTube 전문가 콘텐츠 기반 실전 정보 제공 (신규)
- 408건의 realestate_v3 출처 — 채널명/날짜 인용으로 검증 가능성 제공

**3. 할루시네이션 통제 유지 + 레이턴시 30% 감소**
- 문서 범위 63배 확대에도 halluc pass 84.3% → 84.8% (미세 개선)
- doc pass율 상승 → 재검색 빈도 감소 → 91초 → 64초 단축

## 2. 평가 방법론

### 2-1. 학술적 근거

| 프레임워크 | 출처 | 적용 |
|-----------|------|------|
| **RAGAS** (Shahul Es et al., 2023; v0.2 2024) | Retrieval Augmented Generation Assessment | faithfulness, answer_correctness 개념 차용 |
| **LLM-as-a-Judge** (Zheng et al., NeurIPS 2023) | MT-Bench / Chatbot Arena | 5차원 비교 루브릭 설계 |
| **FActScore** (Min et al., 2023) | Atomic Fact 분해 + 검증 | 할루시네이션 정성 분석 |
| **SAFE** (Google DeepMind, 2024) | Search-Augmented Factuality Evaluator | "정직한 거부" 가치 분석 |
| **Seven Failure Points of RAG** (Barnett et al., 2024) | RAG 실패 유형 분류 | 근본 원인 분석 |

### 2-2. 평가 차원 (5차원 루브릭)

| # | 차원 | 설명 | RAG 차별화 |
|---|------|------|-----------|
| 1 | **사실 정확성** | 규제·세금·대출 사실의 정확도 | document pass 시 우위 |
| 2 | **근거 제시** | 출처/법적 근거 인용 여부 및 품질 | RAG 고유 강점 |
| 3 | **완전성** | 질문의 모든 측면을 다루는 정도 | v3 연동 후 격차 축소 |
| 4 | **실용성** | 사용자가 행동할 수 있는 구체적 조언 | Opus 여전히 우위 |
| 5 | **안전성** | 불확실 시 정직 거부, 면책, 시간 인식 | RAG 우위 |

### 2-3. 분석 방법

- **정량 분석**: `result_v3/summary.md` (v3 연동) 및 `opus_result/summary.md`의 통계 데이터
- **Before/After 비교**: `result/` (v3 미연동) vs `result_v3/` (v3 연동)으로 개선폭 측정
- **카테고리별 비교**: Part A 7개 카테고리(A-1~A-7) + Part C + Part D
- **정성 비교**: 대표 질문의 실제 답변 비교

## 3. 시스템 개요 비교

### 3-1. RAG+LLM 시스템 (Sonnet + realestate_v3)

```
사용자 질의
    │
    ▼
[QueryAnalyzer] — SIMPLE(80%) / REWRITE(16%) / DECOMPOSE(4%)
    │
    ▼
[BGE-M3 Dense+Sparse RRF 검색]
    ├── domain_ontology_v2 (2,146 포인트, 전문 용어)
    ├── legal_docs_v2 (775 포인트, 법률 조문)
    └── realestate_v3 (182,444 포인트, YouTube 전문가 콘텐츠)
    │
    ▼
[Cross-Encoder bge-reranker-v2-m3-ko]
    │
    ▼
[CRAG Compensator] — CORRECT / AMBIGUOUS / INCORRECT
    │
    ▼
[grade_documents] — pass/fail 판정
    │
    ├── pass → [generate] → [check_hallucination] → 답변
    │
    └── fail → [rewrite_query] → 재검색 → [generate] → [check_hallucination]
```

- **검색 범위**: 185,365 포인트 (3개 컬렉션 통합)
- **임베딩 모델**: BGE-M3 (Dense 1024D + Sparse) — 전 컬렉션 동일
- **LLM 호출**: 질문당 평균 ~3.3회

### 3-2. 순수 LLM 시스템 (Opus 4.6)

```
사용자 질의
    │
    ▼
[Claude Opus 4.6] — 파라메트릭 지식만 사용
    │
    ▼
답변 (+ 41% 확률로 후속 제안)
```

- **시스템 프롬프트**: "당신은 대한민국 부동산 전문 상담사입니다."
- **LLM 호출**: 질문당 1회
- **후속 제안**: 41% 자발적 제안, 99.7% 수락

### 3-3. 비용/성능 비교

| 항목 | RAG (v3 연동) | Opus 4.6 | 비교 |

## 5. 카테고리별 상세 분석 (v3 연동 기준)

### 5-1. 세금 (A-2, Q16-30) — RAG 최강 카테고리

v3 연동 후에도 세금 카테고리는 RAG의 핵심 강점 영역이다. legal_docs_v2의 법률 조문 + realestate_v3의 세금 전문 채널(AllTax_GAGAM 637건) 콘텐츠가 결합되어 출처 인용의 깊이가 더해졌다.

#### RAG 출처 인용의 실질적 가치 — Q16, Q27

**Q16: "취득세 얼마 나오나요? 9억짜리 아파트 1주택자요"**

RAG: 법률 문서 기반 3% 세율 계산 + realestate_v3의 세금 전문가 해설 인용
→ 사용자가 법적 근거를 **직접 추적 가능** + 전문가 경험 기반 보충

**Q27: "재산세 6월 1일 기준이라 잔금일 조절해야 한다면서요?"**

RAG: "잔금일과 등기일 중 **더 빠른 날**이 취득일" — 문서 기반 비직관적 인사이트
→ 이 규칙은 문서에서만 정확히 확인 가능

#### 발견된 문제 — Q17 문서 시점 불일치

| 항목 | RAG | Opus |
|------|-----|------|
| 양도세 비과세 기준 | **8억원 이하** (legal_docs_v2 인용) | **12억원 이하** (2021.12 개정 반영) |

→ 법률 문서의 시점 문제. 문서 upload_date 기반 최신성 검증이 필요.

### 5-2. 전세/임대차 (A-4, Q46-60) — RAG 중간 강점

v3 연동으로 임대차 관련 YouTube 콘텐츠(alicehuh 529건)가 추가되어, 기존 법률 문서의 법적 구조 설명 + 실전 경험 조합이 가능해졌다.

**Q52 (확정일자)**: 우선변제권 vs 소액임차인 vs 보증보험 3중 구조 법적 설명 유지
**Q53 (경매 보증금)**: "지역별 소액보증금 기준 상이" — false precision 방지

### 5-3. 매매/시세 (A-1, Q1-15) — v3 연동 후 대폭 개선

**v3 연동 전**: doc pass 낮음, 대부분 "확인할 수 없습니다" 거부
**v3 연동 후**: Part A 전체 doc pass 73% — 시황 분석 YouTube 콘텐츠(maeburitv 1,004건, weolbu_official 1,064건)가 시장 동향/투자 전략 질문에 대한 답변 근거를 제공

다만 **시세 예측/타이밍 판단** 질문("지금 사야 할까요")은 여전히 구조적으로 답변 불가 — 어떤 문서도 미래 시세를 확정적으로 예측할 수 없음.

### 5-4. 대출/청약/경매/지역 (A-3, A-5, A-6, A-7)

v3 연동으로 전 카테고리에서 개선:
- **대출**: 대출 실전 경험 콘텐츠 활용 (DSR/LTV 구체 사례)
- **청약**: 청약 전략/가점 계산 콘텐츠 활용
- **경매**: 경매 전문 채널(Hootv 1,032건) 콘텐츠가 핵심 기여
- **지역**: 지역분석 채널(Busan-apt 438건 등) 콘텐츠 활용

### 5-5. 대화형 (Part D) — v3 연동 후 극적 개선

**Before**: doc pass 4/85 (4.7%), 대부분 "확인할 수 없습니다"
**After**: doc pass **48/85 (56.5%)** — YouTube 콘텐츠가 생활밀착형 질문 커버

| 지표 | RAG v1 (v3 미연동) | RAG v2 (v3 연동) | Opus |
|------|-------------------|-----------------|------|
| doc pass | 4/85 (4.7%) | **48/85 (56.5%)** | N/A |
| halluc pass | 65/85 (76.5%) | 71/85 (83.5%) | 미평가 |
| realestate_v3 출처 | 0건 | **172건** | N/A |
| 멀티턴 컨텍스트 | **0건** | **0건** (미구현) | **12건+** |

> Part D에서 RAG의 **잔여 한계**: 멀티턴 컨텍스트 부재. Opus는 이전 턴을 누적 참조하지만, RAG는 각 턴을 독립 처리. 이는 P3 과제로 식별됨.

## 6. 할루시네이션 분석 (v3 연동 기준)

### 6-1. RAG 할루시네이션 통제: 84.8% pass

| Part | doc pass | halluc pass | 해석 |
|------|---------|------------|------|
| A (100) | 73 (73%) | 89 (89%) | 높은 doc pass → 문서 기반 답변 → halluc 통제 |
| C (12) | 7 (58%) | 7 (58%) | doc pass와 halluc pass 동일 — 문서 부재 시 추가 정보 삽입 |
| D (85) | 48 (57%) | 71 (84%) | doc pass보다 halluc pass 높음 — fail에서도 정직 거부 |
| **전체** | **128 (65%)** | **167 (85%)** | **doc pass < halluc pass = 정직 거부 작동** |

**핵심**: v3 연동 후에도 halluc pass(85%) > doc pass(65%) 패턴 유지. doc fail 69건 중 39건이 halluc pass → "확인할 수 없습니다" 정직 거부가 여전히 작동.

### 6-2. v3 미연동 대비 할루시네이션 변화

| 지표 | Before | After | 변화 |
|------|--------|-------|------|
| halluc pass | 166/197 (84.3%) | 167/197 (84.8%) | +0.5%p |
| halluc fail | 31/197 (15.7%) | 30/197 (15.2%) | -0.5%p |

→ 문서 범위 63배 확대에도 할루시네이션 통제가 완전히 유지됨. check_hallucination 노드가 확장된 문서에도 동일하게 작동.

### 6-3. Opus의 "자신감 있는 부정확" 위험

Opus는 197개 질문에 한 번도 "모르겠다"고 하지 않음. Q17(양도세 비과세 기준 12억 vs RAG의 8억)처럼 검증 불가능한 수치를 자신감 있게 제시하는 위험 존재.

### 6-4. Barnett et al.(2024) 실패 유형 기준 분류

| 실패 유형 | RAG v3 연동 후 | v3 미연동 시 | Opus |
|----------|---------------|-------------|------|
| **Partial Retrieval** | 35% fail → 정직 거부 | 89.3% fail | N/A |
| **Outdated Grounding** | Q17 시점 문제 유지 | 동일 | 학습 cutoff 의존 |
| **Extrinsic Hallucination** | 15.2% | 15.7% | 전체 답변이 파라메트릭 |

→ v3 연동으로 **Partial Retrieval 실패가 89.3% → 35.0%로 대폭 감소** — 가장 큰 개선.

Harvey AI(법률 RAG, 2025): **Recall over Precision** + LLM 필터링

TWICE(ICLR 2025): 한국어 네이티브 벤치마크에서 다국어 모델 번역 대비 5-8% 성능 저하.

| 순위 | 작업 | 기대 효과 | 소요 |

#### P1: document fail 시 fallback 검색 전략

현재 파이프라인은 3개 컬렉션을 **동시에** 검색하지만, grade_documents가 fail 판정 시 **동일한 3개 컬렉션**에서 재검색한다. fail 질문의 상당수는 검색 범위가 아닌 **검색 질의의 표현** 문제이므로, 단계적 전략이 필요하다.

```
[현재]
검색 (onto+legal+realestate) → fail → rewrite → 재검색 (같은 3개 컬렉션)

[제안]
검색 (onto+legal+realestate) → fail → rewrite → 재검색
    → 여전히 fail → realestate 단독 확장 검색 (limit 2배, 메타데이터 필터 완화)
    → 여전히 fail → "확인할 수 없습니다" + 관련 주제 제안
```

- **realestate 단독 확장 검색**: limit을 5→10으로 늘리고, topic_tags/region_tags 기반 필터를 추가하여 검색 recall 확보
- Harvey AI의 "Recall over Precision + LLM 필터링" 전략 적용
- **기대 효과**: 잔여 35% fail 중 초세부 지역(~25%)과 절차적 세부(~15%) 일부 해소, doc pass 65% → ~75%

#### P2: RAG + 파라메트릭 지식 하이브리드 답변

잔여 fail의 ~30%는 "시세 예측/타이밍" 질문으로, **어떤 문서로도 답변 불가**하다. 이 경우 LLM의 파라메트릭 지식을 **조건부로 허용**하되, 명확히 구분 표시한다.

```
[문서 근거 기반 답변]
"취득세는 주택 가격에 따라 1-3%가 적용됩니다. [1] 법률: 제1편 주택의 취득과 관련된 세금"

[일반 지식 기반 보충] ← doc fail 시에만 활성화
"※ 아래 내용은 검색된 문서에 포함되지 않은 일반 정보입니다:
 - 조정대상지역 2주택 시 8% 중과세율이 적용될 수 있습니다.
 - 정확한 적용 여부는 세무사 상담을 권장합니다."
```

- **Retrieval Confidence Scorer**: CE 점수 + CRAG grade 기반 3단계 (HIGH/MEDIUM/LOW)
  - HIGH (CE ≥ 0.7): 확신 있는 답변 + 출처
  - MEDIUM (0.3 ≤ CE < 0.7): 답변 + "참고 수준" 면책
  - LOW (CE < 0.3): 파라메트릭 보완 + "일반 정보" 구분 표시
- Opus의 포괄성과 RAG의 신뢰성을 동시에 확보
- **기대 효과**: 시세 예측 질문에서도 유용한 프레임워크 제공, 사용자 만족도 대폭 향상

#### P3: 멀티턴 컨텍스트 구현

Part D 테스트에서 RAG의 **구조적 한계**: 각 턴을 독립 질의로 처리하여 대화 맥락이 완전히 단절됨. Opus는 동일 세트에서 12건+ 명시적 이전 턴 참조를 보여줌.

```
[현재] 각 턴 독립 처리
Q3: "신촌 원룸 시세?" → 검색 + 답변
Q4: "보증금 1000 월세 50 적당?" → 독립 검색 (Q3의 "신촌" 컨텍스트 무시)

[제안] 대화 히스토리 기반 코레퍼런스 해소
Q3: "신촌 원룸 시세?" → 검색 + 답변 + 컨텍스트 저장
Q4: "보증금 1000 월세 50 적당?" → 코레퍼런스 해소: "신촌 지역에서 보증금 1000 월세 50 적당?"
                                → 컨텍스트 포함 검색 + 답변
```

- `00_overview.md`에서 설계된 코레퍼런스 해소 메커니즘 활용:
  - 슬라이딩 윈도우 5턴
  - 코레퍼런스 마커: "그거", "이거", "그건", "그럼", "거기" 감지
  - 도메인 키워드 부재 + 짧은 질의 → LLM 기반 코레퍼런스 해소 발동
- **기대 효과**: Part D halluc pass 84% 유지하면서 doc pass 57% → ~70% 개선

#### P4: 문서 시점 검증 체계

Q17 사례(양도세 비과세 기준 8억 vs 12억)에서 드러난 문제: **문서가 존재해도 최신이 아닐 수 있음**. 사용자는 RAG의 출처 인용을 "최신 정보"로 신뢰할 가능성이 높다.

해결 방안:
1. 답변 시 문서의 `upload_date` 자동 표시: "이 정보는 {날짜} 기준입니다"
2. 2년 이상 경과 문서에 경고: "이 정보는 {년}년 기준이며, 현행 법령과 다를 수 있습니다"
3. 동일 주제 최신 문서 우선 랭킹 (upload_date 기반 recency bias)

- 수정 파일: `gen_prompts.py` (프롬프트에 시점 인식 규칙 추가), `citation_formatter.py` (출처에 날짜 강조)
- **기대 효과**: false precision 방지, 법률 변경 시 사용자 오해 감소

#### P5: 답변 생성 프롬프트 개선 (Opus 벤치마킹)

Opus 답변의 강점을 RAG 프롬프트에 흡수:
- **구조화된 답변 포맷**: 표, 체크리스트, 단계별 가이드 (Opus 스타일)
- **실용적 행동 지침**: "지금 당장 할 수 있는 행동 3가지" 섹션
- **맥락 적응**: 질문자의 상황(신혼부부, 사회초년생, 투자자)에 맞는 어조
- **후속 제안**: Opus의 41% 후속 제안처럼, 관련 주제 2-3개 자발적 제안

현재 `GENERATE_SYSTEM_PROMPT`에 추가:
```
- 답변은 표, 비교 차트, 단계별 가이드 형태로 구조화하세요
- 답변 끝에 "추가로 궁금하실 수 있는 관련 주제" 2-3개를 제안하세요
- 질문자의 상황(초보/전문가, 실거주/투자)을 추론하여 어조를 조절하세요
```

- **기대 효과**: 답변 사용자 만족도 향상, Opus 대비 "완전성"과 "실용성" 차원 격차 축소
